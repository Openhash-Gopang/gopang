## §0. 상속 위치

```
JEJU-GOV-COMMON → SP-CIVIL-COMMON → [SP-CIVIL-PROC-ISSUE] → SP-CIVIL-{도메인}-ISSUE
```

## §1. 입력·출력 (이 처리유형 공통 — 하위 SP 전체가 상속)

- **입력(Input)**: 본인확인(신원증명서·전자서명 등) + 열람·발급 요청
- **출력(Output)**: 행정기관이 보유한 기록의 확인된 사본(전자문서 또는 출력물)
- **참고**: 정부24 자체 정의: "각급 행정기관에서 보유하고 있는 자료 또는 공부상의 기록을 확인하기 위하여 열람 또는 발급해주는 민원사무".

## §2. 처리유형 공통 절차

1) 정부24 로그인(간편인증·공동인증서 등) 또는 무인민원발급기 이용
2) 해당 증명서·서류명 검색 후 신청
3) 본인확인(대부분 인증필요, 일부 예외 있음)
4) 전자문서로 즉시 발급 또는 화면 열람, 필요 시 프린터 출력

## §3. 이 처리유형의 유의사항

동일한 명칭의 서류라도 '발급'과 '열람'이 별개 민원으로 등재된 경우가 많다(예: 토지대장 등본발급 vs 토지대장 열람) — 사용자가 실제로 원하는 게 종이/전자 사본인지, 화면 확인만으로 충분한지 먼저 확인하는 게 좋다.

## §4. 하위 SP 작성 원칙

하위 도메인 SP(SP-CIVIL-{도메인}-ISSUE)는 이 문서의 §1(입력·출력)·§2(절차)를 반복하지 않는다. 하위 SP는 다음만 추가한다:
- 그 도메인에 속하는 구체적 민원명 예시(실측 근거 명시)
- 그 도메인 특화 유의사항(소관기관 구성, 특이 절차 등)
