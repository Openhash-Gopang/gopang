## §0. 상속 위치

```
JEJU-GOV-COMMON → SP-CIVIL-COMMON → [SP-CIVIL-PROC-INQUIRY] → SP-CIVIL-{도메인}-INQUIRY
```

## §1. 입력·출력 (이 처리유형 공통 — 하위 SP 전체가 상속)

- **입력(Input)**: 본인확인(또는 공개정보인 경우 불필요)
- **출력(Output)**: 조회된 정보(화면 표시, 경우에 따라 인쇄 가능)
- **참고**: 정부24 통계 정의상 "신청하고 신청결과를 PC화면으로 확인한 건수"를 열람으로 별도 집계한다.

## §2. 처리유형 공통 절차

1) 조회 대상 정보가 본인정보인지 공개정보인지 확인
2) 본인정보라면 인증 필요, 공개정보라면 바로 조회 가능
3) 화면상 정보 확인 — 발급형과 달리 공식 증명서로서의 효력이 없는 경우가 많음(단순 확인용)

## §3. 이 처리유형의 유의사항

조회 결과가 '공식 증명서'로 쓸 수 있는지, 아니면 '참고용 확인'일 뿐인지 구분해서 안내해야 한다 — 제출용 서류가 필요하면 조회가 아니라 발급(ISSUE)형으로 다시 안내해야 할 수 있다.

## §4. 하위 SP 작성 원칙

하위 도메인 SP(SP-CIVIL-{도메인}-INQUIRY)는 이 문서의 §1(입력·출력)·§2(절차)를 반복하지 않는다. 하위 SP는 다음만 추가한다:
- 그 도메인에 속하는 구체적 민원명 예시(실측 근거 명시)
- 그 도메인 특화 유의사항(소관기관 구성, 특이 절차 등)
