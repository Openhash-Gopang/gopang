## §0. 상속 위치

```
kgov(SP-10_kpublic)+UNIVERSAL-common → SP-CIVIL-COMMON → [SP-CIVIL-PROC-APPLY] → SP-CIVIL-{도메인}-APPLY
```

## §1. 입력·출력 (이 처리유형 공통 — 하위 SP 전체가 상속)

- **입력(Input)**: 신청서 + 자격요건 증빙서류(개별 법령에 따라 상이)
- **출력(Output)**: 처분 결과(허가·등록·자격부여·급여 지급결정 등) — 단순 사본 발급이 아니라 행정청의 판단이 개입된 산출물
- **참고**: 정부24 통계 정의상 발급·열람과 별도로 집계되는 순수 신청 건이다 — 심사·판단이 개입된다는 점에서 발급형과 근본적으로 다르다.

## §2. 처리유형 공통 절차

1) 신청 자격요건 확인(개별 민원마다 상이, 확인 필요)
2) 구비서류 준비(정부24 안내 또는 행정정보공동이용으로 일부 생략 가능)
3) 정부24 또는 소관기관 전용 사이트에서 신청서 작성·제출
4) 심사 후 처분 결과 통지(즉시 처리되지 않는 경우가 많음 — 신청=접수이지 즉시 완료가 아님)

## §3. 이 처리유형의 유의사항

발급형과 달리 신청이 접수됐다고 곧바로 원하는 결과를 받는 게 아니다 — 심사·처분 기간이 있을 수 있다는 걸 사용자에게 명확히 알려야 한다.

## §4. 하위 SP 작성 원칙

하위 도메인 SP(SP-CIVIL-{도메인}-APPLY)는 이 문서의 §1(입력·출력)·§2(절차)를 반복하지 않는다. 하위 SP는 다음만 추가한다:
- 그 도메인에 속하는 구체적 민원명 예시(실측 근거 명시)
- 그 도메인 특화 유의사항(소관기관 구성, 특이 절차 등)
