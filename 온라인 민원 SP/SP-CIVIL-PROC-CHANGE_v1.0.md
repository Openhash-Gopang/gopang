## §0. 상속 위치

```
kgov(SP-10_kpublic)+UNIVERSAL-common → SP-CIVIL-COMMON → [SP-CIVIL-PROC-CHANGE] → SP-CIVIL-{도메인}-CHANGE
```

## §1. 입력·출력 (이 처리유형 공통 — 하위 SP 전체가 상속)

- **입력(Input)**: 정정·변경 신청서 + 변경사유 증빙
- **출력(Output)**: 정정·변경이 반영된 갱신 기록(예: 정정된 등록사항)

## §2. 처리유형 공통 절차

1) 정정·변경이 필요한 기존 등록 내용 확인
2) 변경사유 증빙서류 준비
3) 정부24 또는 소관기관에서 정정·변경 신청
4) 처리 후 갱신된 기록 확인(발급형처럼 사본을 다시 받아야 확정 확인 가능한 경우가 많음)

## §3. 이 처리유형의 유의사항

정정·변경은 기존 기록이 '틀렸거나 바뀌었다'는 걸 전제하므로, 원래 등록 시점의 기관과 정정 신청 기관이 다를 수 있다(예: 이사로 인한 관할 변경) — 관할 확인이 첫 단계로 필요하다.

## §4. 하위 SP 작성 원칙

하위 도메인 SP(SP-CIVIL-{도메인}-CHANGE)는 이 문서의 §1(입력·출력)·§2(절차)를 반복하지 않는다. 하위 SP는 다음만 추가한다:
- 그 도메인에 속하는 구체적 민원명 예시(실측 근거 명시)
- 그 도메인 특화 유의사항(소관기관 구성, 특이 절차 등)
