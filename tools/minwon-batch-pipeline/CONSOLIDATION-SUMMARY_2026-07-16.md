## 1. 이번 정리에서 변경한 것

### 1-1. taxonomy 확정 (17개 → 20개)

3배치 누적 검증 결과, 아래 3개 도메인이 반복 확인돼(각 2회 이상) 정식 도메인으로 편입했다.
더 이상 "17개 밖 후보"로 재보고하지 않는다.

| 도메인 | 확인 횟수 | 근거 |
|---|---|---|
| `IDENTITY_REG`(주민등록·신원) | 4회 | 배치1 3건 + 배치2 1건 + 배치3 1건 |
| `PUBLIC_SAFETY`(치안·신고·분실습득) | 3회+ | 배치1 1건 + 배치2 1건(습득조회) + 배치3 2건(고소고발·여권분실) |
| `RESERVATION`(예약형) | 2회 | 배치1(자동차검사예약) + 배치3(출입국 방문예약) |

**아직 편입 보류(관찰 대상, 1회만 관찰)**: `CERT_EXAM`(자격검정, 배치2 #35 2급생활스포츠지도사),
`POLICY_INFO`(제도 안내형, 배치2 #43·배치3에서도 유사 패턴). 다음 배치에서 재확인되면 편입.

### 1-2. R4(관문서비스 탐지) 정밀도 개선

3배치 누적 결과 설명문 단독 키워드("통합이력"·"한 번에 쉽고 간편하게"·"일괄정정" 등)만으로
판단하면 오탐이 반복됐다(배치2 #19·#30, 배치3 #43 — 전부 실제로는 단일 서비스였음). 신뢰도를
2단계로 분리:
- **high**: 제목 자체가 "통합처리/통합제공/일괄신청" 등을 명시적으로 표방(3배치 동안 오탐 0건).
- **low**: 설명문에만 약한 키워드 존재 — 검토자가 노이즈를 감안할 것.

재실행 결과 배치2·3에서 각각 저신뢰 2건·1건이 분리됐고, 정확히 기존 오탐 사례(#19, #30, #43)와
일치했다.

### 1-3. 신설 규칙 (R8) 및 확장(R3 배치 간)

- **R8**: 전국 표준형 서비스인데 특정 시군구 하나만 소관기관으로 표기된 경우 검증 요청.
  3배치에서 3건 확인(배치2 #26 충청북도 보은군, 배치2 #39 전남광주통합특별시 영광군,
  배치3 #47 충청남도 홍성군) — 원본 데이터가 전국 표준 서비스를 수집할 때 특정 지자체
  사례 하나만 대표로 가져오는 방식임을 구조적으로 확정.
- **R3 배치 간 확장**: "문서군(document family)" 패턴을 배치를 넘어 탐지. 확인된 문서군:
  가족관계증명서 계열(혼인관계·영문·입양관계·친양자입양관계, 4건, 2개 배치) +
  검정고시 합격증명서 계열(국문·대입전형용, 2건, 2개 배치).

## 3. 반복 확인된 구조적 패턴 (원본 데이터의 체계적 특성 — 오류 아님/오류인 것 구분)

**오류로 확정된 패턴**:
- 주민등록 계열(`IDENTITY_REG`)이 "주택 및 부동산"으로 오분류 — 4회 반복, 우연 아님.
- 마지막 항목(#50)의 버튼 데이터 누락 — 3배치 연속, 원본 문서 추출의 말단 절단으로 추정.

**오류가 아니라 원본 데이터의 특성으로 확인된 패턴**:
- 지자체별 동명 사업 다수 존재(혜택 카탈로그 쪽과 유사 패턴 — 다만 이쪽은 발급형이라 빈도는 낮음).
- 전국 표준 서비스를 특정 지자체 사례 하나로 수집(R8, 3건 확인).
- 행정구역명은 이미 최신(전남광주통합특별시 반영, 옛 이름 잔존 없음).
