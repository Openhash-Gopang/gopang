혼디, 하루의 풍경

가게 간판에 적힌 색색의 글자 하나가 주문이 되고, 민원이 되고,
다음 사람의 편리함으로 이어지는 과정을 네 장면으로 보여드립니다.

어떻게 색깔이 주소가 되나요?

"혼디" 글자의 마지막 점, "ㅣ" 안에는 작은 색깔 칸들이 쌓여 있습니다. 색깔마다 정해진 숫자가 있어서, 칸을 위에서 아래로 읽으면 하나의 긴 번호가 만들어집니다. 이 번호는 오직 그 가게(또는 사람·기관)의 혼디 주소 하나와만 정확히 연결되어 있습니다 — 전화번호나 우편번호처럼, 번호 하나에 장소 하나가 정해져 있는 것과 같은 원리입니다. 그래서 카메라로 이 색을 읽기만 하면, 번호를 거쳐 곧바로 그 주소(프로필)로 이동하게 됩니다.

ㅎ · ㅗ · ㄴ · ㄷ — 색을 정확히 읽기 위한 "기준색"입니다. 조명이 어둡거나 화면이 누렇게 보여도, 미리 알고 있는 이 기준색과 비교해서 원래 색을 되살립니다. 마치 사진을 찍을 때 화이트밸런스를 맞추는 것과 같습니다.

마지막 점 "ㅣ"의 색칸들 — 실제 주소를 담고 있는 부분입니다. 칸 하나하나가 숫자 하나이고, 칸을 다 읽으면 그 가게만의 고유한 번호가 됩니다.

TECHNICAL SPECIFICATION

혼디 색상 코드 — 기술 명세

1. 색상 팔레트 (9진법)

각 색상은 0~8의 숫자 하나에 대응합니다. 무색(흰색)도 하나의 값으로 취급되어, 총 9개 상태로 한 칸의 숫자를 표현합니다.

값색상HEX
0무색(흰색)#FFFFFF
1빨강#E5293C
2주황#F08A1C
3노랑#F4D61E
4초록#2FA84F
5파랑#1E4FD8
6남색#3D1E7A
7보라#7B2FB5
8흑색#161616

팔레트 인접 색 사이의 RGB 유클리드 거리를 최소 100 이상으로 확보해, 백열등·역광 등 색온도 왜곡 환경에서도 오인식을 줄였습니다. 특히 주황/노랑처럼 거리가 가까운 쌍은 톤을 추가로 분리했습니다.

2. 인코딩 — short_id → 색상 배열

  • 등록 시 발급된 short_id를 9진법으로 변환하고, 각 자리수를 위 팔레트에 매핑합니다.
  • 변환된 숫자열을 "ㅣ" 자모의 칸에 위에서 아래로 채웁니다.
  • v1 (초기 버전) — 세로 10칸, 단열 구성 → 9^10 = 3,486,784,401개 주소 표현
  • v2 (확장 버전) — 세로 10칸 × 좌우 2열(총 20칸) → 9^20 ≈ 1.2 × 10^19개 주소 표현. "ㅣ"의 가로폭은 v1과 동일하게 유지됩니다.
  • 버전 판별은 "ㄷ" 자모의 색으로 이루어집니다 — 단색 흑색이면 v1, 흑색과 회색의 조합이면 v2로 인식합니다.

3. 캘리브레이션 (기준점)

  • ㅎ = #1E4FD8(파랑) — 청색 채널 보정 기준
  • ㅗ = #E5293C(빨강) — 적색 채널 보정 기준
  • ㄴ, ㄷ = 흑색 — 암부(어두운 영역) 보정 기준, 이중 샘플링으로 검증
  • 배경 = 흰색 — 명부 및 화이트밸런스 보정 기준
  • 보정 계수는 채널별로 실제값 = 측정값 × (기준색 채널값 / 측정된 기준색 채널값)으로 계산되어, 데이터 칸의 색상 추출 전에 적용됩니다.

4. 인식 파이프라인

  • 카메라 프레임을 1/4 크기 썸네일로 축소해 "혼디" 글자 윤곽을 1차 탐지 (속도 최적화)
  • 종횡비 검증(1.4~3.5 범위)으로 오탐지 제거 후, 원본 해상도에서 정밀 좌표 재추출
  • ㅎ·ㅗ·ㄴ·ㄷ·배경 샘플링 → 캘리브레이션 보정 계수 산출
  • ㄷ 색상 분석으로 v1 / v2 판별
  • "ㅣ" 영역을 칸 단위로 분할, 보정된 색상 값을 팔레트 9색 중 최근접색에 매칭(최소 색상거리 ≥ 100 검증)
  • 9진수 → 10진수 변환으로 short_id 복원
  • L1 PocketBase 조회 → Ed25519 서명 검증 → GUID(IPv6) 확인 → 해당 프로필로 라우팅
  • 연속 4프레임 동일 결과 시 인식 확정(LOCK_FRAMES=4, 저조도 환경에서는 6으로 상향), 진동·비프음 피드백
  • 보조 모드: 정지 사진 1장을 5회 반복 샘플링해 평균값으로 디코딩 — 저조도·흔들림 환경에서 연속 스캔보다 안정적

5. 주소 공간

버전칸 구성표현 가능 주소 수
v1세로 10칸 (단열)9¹⁰ = 3,486,784,401
v2세로 10칸 × 2열 (20칸)9²⁰ ≈ 1.2 × 10¹⁹

혼디는 출시 시점부터 v1으로 시작하며, 주소 공간이 소진되는 시점에 v2로 전환됩니다. 두 버전 모두 카메라가 자동으로 판별하므로 사용자가 별도로 버전을 선택할 필요는 없습니다.

6. 보안

색상 코드가 가리키는 값은 그 자체로는 의미가 없는 식별자(short_id)일 뿐입니다. 실제 프로필 연결은 L1 PocketBase에 등록된 Ed25519 서명 검증을 통과해야만 이루어지므로, 코드 이미지를 그대로 복제하더라도 서명 없이는 위조된 프로필로 연결될 수 없습니다.
