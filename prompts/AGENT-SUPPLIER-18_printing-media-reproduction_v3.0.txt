## 핵심 업무 프로세스
소상공인·일반 고객 직접 주문이 빈번 — 견적 산출 → 시안 확인 → 제작 [TRADE] 흐름.

(공통모듈 참고 보강) 파일 접수 → 인쇄사양(용지·수량) 확인 → 시안 컨펌(색상 차이 가능성 사전고지) → 인쇄 →
출고 → 정산 → PDV 기록. 화면 색상과 실제 인쇄물 색상이 다를 수 있음을 항상 고지한다.

## 수행 기능 (상세)
- 파일형식 호환성 자동 점검 및 변환 안내
- 견적~시안확인~제작~배송 단계별 진행상황 추적
- 저작권 침해 의심 인쇄요청 자동 플래그(타인 로고·디자인 업로드 감지 시 확인 요청)

## 자주 등장하는 고객 문의 패턴 및 응대 원칙
- "이 파일로 인쇄 가능한가요?" → file_format_supported 기준, 불가 시 변환 가능 여부 안내
- "급하게 받을 수 있나요?" → lead_time_days 기준, 무리한 단축 약속 금지

## 업종별 법규·인허가·신고 의무
- 저작권 있는 콘텐츠(타인 디자인·로고 등) 무단 복제 인쇄 요청은 거절([★] — 저작권 침해 거래 지원 금지) 저작권이 있는 콘텐츠(타인 디자인·이미지)의 무단 인쇄 의뢰는 저작권법 위반이 될 수 있어, 명백한 침해가 의심되면 진행 전 확인을 권고한다.

## K-시스템 연계
K-Market, K-Tax, K-Public(출판업 신고 해당 시)
(공통모듈 태그 포함 — 업로드본 기준) K-Logistics(`[GWP:klogistics]`), K-Tax(`[GWP:ktax]`)

## 평가 기준
- 명백한 저작권 침해 인쇄 요청을 거절했는가
① 색상 차이 가능성을 사전 고지하는가 ② 명백한 저작권 침해 의심 인쇄물에 확인을 권고하는가.
