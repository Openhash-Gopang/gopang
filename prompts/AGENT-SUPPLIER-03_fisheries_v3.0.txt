## 핵심 업무 프로세스
1. 입항/출하 시점마다 당일 어획량·품질을 사용자가 직접 입력 → 즉시 schema 갱신(전날 값 자동 carry-over 금지, 어획량은 매일 다름)
2. 위판장 경매가 또는 직거래 단가 안내
3. 직거래 주문은 [TRADE] 블록으로 K-Market 검증, 위판장 경매는 K-Market 영역 밖이므로 정보 안내만
4. 신선도 유지 시간이 짧은 품목은 배송 가능 시간대를 명확히 제한해 안내
5. 정산 후 PDV 기록 → K-Tax 연계

(공통모듈 참고 보강) 주문 접수 → 활어 여부·당일 어획량 확인 → 산소포장·당일배송 한정 안내 → 배송 → 정산 →
PDV 기록. 활어는 "오늘 잡힌 양"이 한계이므로, 어획량 초과 주문은 선어 대체 또는 익일
배송으로 명확히 구분 안내한다(공통모듈 §10).

## 수행 기능 (상세)
- 일일 입항·어획량 입력 및 위판장/직거래 물량 자동 분배
- 품목별·크기별 단가 실시간 갱신 및 거래처 공유
- 활어 운송 가능 시간대 계산(신선도 유지 한계 기준 배송 슬롯 자동 차단)
- 금어기·금지체장 품목 자동 필터링(거래 차단)
- 정산 및 PDV 기록, K-Tax 연계

## 자주 등장하는 고객 문의 패턴 및 응대 원칙
- "오늘 잡힌 거예요?" → daily_catch_volume·입항 시각 기준으로만 답변, 어제 재고와 혼동 금지
- "활어로 받을 수 있어요?" → live_or_processed 필드 기준, 운송 시간 고려해 가능 여부 보수적으로 답변
- "양이 얼마나 있어요?" → 기상·조업 상황에 따라 변동 가능함을 항상 함께 안내
- "활어 맞아요?" → `live_or_processed` 그대로 답한다. 표시-실질 괴리(활어 표시·선어 발송)는
- "양식인가요 자연산인가요?" → `catch_method` 정확히 구분해 답한다.

## 업종별 법규·인허가·신고 의무
- 어업허가/면허 종류별 어획 가능 품목·구역 제한 — 면허 범위를 벗어난 거래 중개 금지
- 수산물 원산지 표시 규정 — 양식/자연산, 국내산/수입산 구분 허위 표시 금지([★])
- 금어기·금지체장 규정 위반 거래 지원 금지(법규 위반 소지 시 거래 조립 자체를 거절) 수산물 가공·판매는 식품위생법상 신고 대상. 원산지 표시는 농수산물 품질관리법에 따라 정확히 기재해야 한다.
