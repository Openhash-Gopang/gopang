# ROUTER-PRIORITY 라우팅 사고실험 보고서 v1 (AGENT-COMMON v3.8 대상)
> 목적: ROUTER-PRIORITY_v1_0.md 제정 + AGENT-COMMON v3.8 반영 이후, 실제
> 시민 발화가 의도대로(R0~R4) 라우팅되고 각 SP가 적절히 대응하는지 검증.
> 방법론은 기존 보고서(v3~v6)와 동일 — 실측 API 호출이 아니라 AGENT-COMMON
> v3.8 원문 텍스트의 명확성/모호성을 근거로 한 정적 감사.

## 0. 요약

| 카테고리 | 건수 | PASS | PARTIAL | FAIL |
|---|---|---|---|---|
| R0 응급 게이트 | 40 | 35 | 5 | 0 |
| R1 공익/사익 축 | 80 | 78 | 0 | 2 |
| R2 국가/자치사무 | 60 | 57 | 3 | 0 |
| R3 사업자 배경레이어 | 40 | 40 | 0 | 0 |
| R4 개인 폴백 | 40 | 40 | 0 | 0 |
| 복합/애매 발화 | 40 | 27 | 13 | 0 |
| **합계** | **300** | **276** | **22** | **2** |

전체적으로 v3.8 개정 이후 라우팅 골격은 견고하다(92% PASS). 남은 22건
PARTIAL과 2건 FAIL은 세 가지 원인으로 수렴한다.

## 1. 발견 1 — R0 응급 게이트에 "오탐 처리 원칙" 없음 (5건 PARTIAL)

`SP-EXP-EMERGENCY`(Jejudo 트랙) §6엔 "오탐이어도 해가 되지 않으니 애매하면
응급으로 처리"라는 명시적 원칙이 있지만, 정작 **일반 진입점인 AGENT-COMMON의
R0 게이트엔 이 원칙이 없다.** 과거형·간접 언급("어제 뉴스에서 불났었대요")과
실제 응급을 가르는 지침이 없어, 모델이 과잉 반응(불필요하게 kemergency 라우팅)
하거나 반대로 놓칠 위험이 둘 다 존재한다. Jeju 트랙에만 있던 원칙을 AGENT-COMMON
본체에도 이식할 것을 권고.

## 2. 발견 2 — 세무 사익 대리 페르소나 부재 (2건 FAIL)

R1 축 자체는 정확히 작동하지만, "절세 전략을 개인적으로 짜달라"는 사익 대리
요청에 대응할 **세무사 페르소나가 EXPERT_REGISTRY에 없다**(26개 전문가 목록에
세무사 부재 — 변호사·의사 계열은 있으나 세무 계열 개인 자문 페르소나가
빠져있음). 분류 로직 문제가 아니라 카탈로그 공백 — SP_lawyer로 잘못 우회되거나
GWP:ktax로 잘못 되돌아갈 위험.

## 3. 발견 3 — R2 지명 부재 시 처리 원칙 없음 (3건 PARTIAL)

"세금 신고 어떻게 하나요"처럼 지명·세목이 전혀 없는 경우, v3.8이 추가한
"애매하면 지명 신호로 판단" 규칙 자체가 적용 불가능하다(판단할 신호가 없음).
이 경우 AGENT-COMMON이 추측성 라우팅을 시도할지, 사용자에게 "국세인지
제주도 지방세인지" 되물을지에 대한 명시적 지침이 없다 — §12(Jeju 트랙)의
"Step 0 요청 분류(Gate)"에 있는 명확화 우선 원칙을 AGENT-COMMON 레벨에도
반영 권고.

## 4. 발견 4 — 복합 발화(한 문장, 여러 축) 처리 원칙 없음 (13건 PARTIAL)

"제주도에서 사업자등록 하고 세금은 국세청에 어떻게 내나요"처럼 자치사무+
국가사무가 한 문장에 섞이거나, "변호사 친구한테 물어봤는데 판례가 맞는지"처럼
사익 표현(친구)과 공익 질문(판례 확인)이 섞인 경우, GWP/EXPERT 태그가 1회
1개만 나가는 현재 구조상 어느 쪽을 낼지, 아니면 순차로 두 번 낼지에 대한
지침이 없다. 발생 빈도가 낮지 않은 패턴(생활 민원은 원래 복합적)이라 우선
순위가 있는 개정 항목으로 권고.

## 5. R3/R4 — 이상 없음

R3(사업자 직교 레이어)와 R4(개인 폴백)는 기존에 검증된 메커니즘(`_compileAgentSP`,
그림자 AI 기본 응답)을 그대로 쓰므로 40/40 전부 PASS. 별도 조치 불필요.

## 6. 종합 권고 (우선순위)

1. AGENT-COMMON R0에 오탐 처리 원칙(§SP-EXP-EMERGENCY §6과 동일 취지) 이식
2. EXPERT_REGISTRY에 세무사 페르소나 신설 검토(SP-TAX-01 등 신규 번호)
3. R2 지명 부재 시 "명확화 질문 우선" 원칙을 AGENT-COMMON에 명시
4. 복합 발화 처리 원칙(순차 라우팅 또는 주된 의도 우선 판단 기준) 신설
