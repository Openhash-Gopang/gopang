## 1. 방법론과 자기 교정 — 왜 40건이 7건으로 줄었나

각 SP 파일에서 정규식 키워드(`정직`, `지어내지`, `미검증`, `모르면 모른다`,
`VERIFY`, `PDV_REQUEST`, `승인`, `C23`, `면허`, `119` 등)를 검색해 원칙
보유 여부를 1차 판정했다. 이 방식은 **"이 파일 자체에 그 단어가 있는가"만
보고, 상위 공통문서에서 상속하는 원칙은 놓친다** — 지금까지 여러 라운드에서
반복 확인된 함정과 같은 종류다.

1차 스캔에서 "데이터연동공백(정직성)" 카테고리 40건이 FAIL로 나왔다. 이걸
그대로 보고하지 않고 교정했다:

- **정부 5건**(JEJU-DO-SP, SP-CITY-JEJU/SEOGWIPO, SP-EMD-TEMPLATE/HALLIM):
  JEJU-GOV-COMMON §10을 구조상 상속 — false positive → PASS
- **전문직 18건**: `expert-session.js`가 `SP_common_guardrails`(C24 지식한계
  고지)를 전 페르소나에 코드로 prepend하는 게 이미 검증돼 있음 — false
  positive → PASS
- **K-서비스 9건**: `GOV_AGENCIES` 9개 서비스는 `K-Public_common_v1_1.md`
  P3(정직성)를 상속(배포 완료 확인됨) — false positive → PASS
- **K-Law 1건**: 직접 열어보니 "정직"이라는 단어는 없지만 **훨씬 정교한
  자체 메커니즘**을 보유 — 법리·사실 확신도를 0~10으로 이원 채점하고,
  불확실한 판례번호는 생성 자체를 차단하며("대법원 XXXX다XXXXX" 패턴
  감지 시 즉시 원칙수준 서술로 치환), 허위 판례 적발 시 최대 -3.0점
  페널티를 부과한다. 내 키워드 검색이 이 정교함을 못 잡아낸 것 — PASS로
  교정.
- **나머지 7건**: 교정 후에도 FAIL 유지 (§2 참조)

## 3. PARTIAL 141건 — 새로운 결함이 아니라 기존 패턴의 재확인

"복잡수행요구"·"권한경계테스트" 카테고리에서 개별 SP 파일 자체에 실행
예시·권한경계 문구가 없는 경우 PARTIAL로 표시했다. 이 역시 상위 공통문서
상속으로 원칙 자체는 적용될 가능성이 높지만(§1과 같은 이유로 일부는
false positive일 수 있음), "원칙은 있으나 그 SP 도메인에 특화된 실증
예시가 없다"는 기존 v3 보고서의 핵심 권고(도메인별 실행 예시 보강)가
94개 전체로 확장돼도 여전히 유효하다는 뜻이다. 개별 141건을 전수
재검증하진 않았다 — 우선순위상 §2의 실제 FAIL 7건 해결이 먼저다.
