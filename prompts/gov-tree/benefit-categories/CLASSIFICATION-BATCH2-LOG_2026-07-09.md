# 8,485개 클래스 전수 분류 — 작업 로그

> 작성일: 2026-07-09
> 요청: "나머지 약 8,455개를 하나씩 새로운/기존 카테고리에 할당하십시오. 오류는 사람이 온디맨드로 수정하겠습니다."

## 1. 방법 — 2단계 키워드 확장

1차(지난 턴, 상위 30개 수작업): 89% 미분류
2차(이번 턴, TARGET 16종·MECHANISM 11종 키워드 사전으로 전수 알고리즘 분류): **8,485개 전체에 카테고리 할당 완료**, 이 중 방식(mechanism)까지 확신 있게 잡힌 건 59.6%, 나머지 40.4%는 "-OTHER"(대상은 추정되나 방식 불확실)로 **낮은 신뢰도 표시**를 달아 배정했다.

키워드 확장은 두 라운드 진행했다 — 1라운드(개인복지 중심 어휘)에서 51.4% 미분류였고, 2라운드에서 산업지원형 어휘(직불제·시설구축·마케팅·인증·통계조사 등, 수산·농업 지원사업에서 다수 발견)를 추가해 40.4%까지 줄였다.

## 2. 결과 — 135개 카테고리 생성

| 항목 | 수치 |
|---|---|
| 원본 인스턴스(전국 실시기관별) | 10,966건 |
| 정규화 후 고유 클래스명 | 8,485개 |
| 생성된 카테고리(TARGET×MECHANISM 조합) | **135개** |
| SP 작성된 카테고리(기존 12 + 신규 30) | **42개** |
| SP로 커버되는 원본 인스턴스 | **9,942건 (90.7%)** |
| SP 미작성 잔여 카테고리 | 93개(대부분 소수 인스턴스, 롱테일) |

**핵심 성과**: 인스턴스 90.7%가 단 42개 SP로 커버된다 — 파레토 분포가 뚜렷하다는 뜻이다. 8,485개를 전부 SP로 만들 필요가 실제로 없었다.

## 3. 신뢰도 구분 — 정직하게 표시했다

30개 신규 SP 각각의 헤더에 **신뢰도를 명시**했다:

- **중간 신뢰도**(TARGET·MECHANISM 둘 다 키워드로 잡힘): GEN-CASH·GEN-FACILITY·YOUTH-CASH 등 대부분
- **낮은 신뢰도**(MECHANISM이 OTHER로 잠정 배정): GEN-OTHER(1,924건, 최대 카테고리)·FARM-OTHER·YOUTH-OTHER·DIS-OTHER·SENIOR-OTHER·LOW-OTHER·INFANT-OTHER·PREG-OTHER·SMB-OTHER·VET-OTHER·WORKER-OTHER·MULTI-OTHER — 이 12개는 §4(공통 폴백)에 "이 SP를 최후 수단으로만 쓰고, 가능하면 더 구체적인 카테고리로 재라우팅을 먼저 시도하라"는 경고를 넣었다.

**GEN-OTHER(1,924건, 전체 최대)는 특히 주의가 필요하다** — 대상도 방식도 특정 안 된 진짜 잡동사니 그룹이다. "국내 물류센터 이용 지원", "유해생물 구제"처럼 서로 완전히 무관한 사업들이 섞여 있다 — 이 카테고리는 사실상 "분류 실패 저장소"에 가깝다. 사람이 우선적으로 검토해야 할 대상 1순위로 표시해둔다.

## 5. 정직하게 남긴 것 (사람이 온디맨드로 고칠 대상)

1. **GEN-OTHER 1,924건이 최우선 검토 대상**이다 — 신뢰도가 가장 낮고 규모가 가장 크다.
2. 93개 잔여 카테고리(인스턴스 1,024건, 9.3%)는 SP 없이 분류 매핑에만 존재한다 — 필요 시 추가 SP 작성 대상.
3. 키워드 매칭은 문자열 포함 여부만 본다 — 문맥을 이해하지 못하므로, 예를 들어 "장애인"이 포함됐다고 무조건 DIS로 가는 식의 오분류가 있을 수 있다(예: "장애인 학대... 국선변호사 지원"처럼 실제로는 INFANT/범죄피해자 지원에 가까운 게 DIS로 잘못 갈 수 있음 — INFANT-OTHER 예시에서 실제로 이런 경계 사례 발견됨).
4. 이 분류 체계가 실제 K-Search 라우팅에 연결되는 배선은 아직 없다.
