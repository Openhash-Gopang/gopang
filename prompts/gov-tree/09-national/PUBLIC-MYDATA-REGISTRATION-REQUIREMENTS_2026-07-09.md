# 공공 마이데이터 이용기관 등록 요건 — Hondi 적용 가능성 평가

> 작성일: 2026-07-09
> 배경: "AI 비서가 실제로 증명서를 발급받을 수 있어야 한다"는 요구에 대한 답으로, 정부24 UI 자동화나 행정정보공동이용(기관간 전용)이 아니라 **공공 마이데이터(mydata.go.kr)**가 민간 서비스도 이용기관으로 등록 가능한 정식 경로임을 확인했다. 등록 요건을 파고들었다.

---

## 1. 용어 정리 (혼동 방지)

| 용어 | 의미 |
|---|---|
| 보유기관 | 행정정보를 원래 갖고 있는 정부기관(예: 행정안전부=등본 보유) |
| **이용기관** | 본인정보를 제공받는 제3자 — **Hondi가 되려는 자리** |
| 이용지원기관 | 여러 이용기관을 지원하는 행정·공공기관(Hondi와 무관) |
| 전자문서지갑 | 사용자가 받은 본인정보를 보관하는 개인용 전자공간 |
| 보안저장소 | 이용기관이 받은 본인정보를 보관하는 **기관측** 저장소 (Hondi가 구축해야 하는 것) |
| 공공 마이데이터 전담관리자 | 이용기관 소속, 마이데이터 업무를 총괄하는 담당자(1인 이상 지정 필요) |

---

## 2. 등록 절차 — 5단계

### 1단계: 이용기관 신청 (서류 5종)

행정안전부 행정정보공유과에 아래 5개 문서를 공문으로 제출(+ helpdesk@mydata.go.kr 이메일 병행 제출 권장):

1. 이용기관 신청서
2. 이용 환경 조사표
3. 묶음정보 이용·해지 신청서
4. 묶음정보 정의서(샘플)
5. 업무구조도(샘플)

**"묶음정보"**란 여러 종류의 본인정보를 하나의 서비스 목적으로 묶은 단위다 — 예를 들어 Phase 1에서 다룬 25건 중 개인파산 절차에 필요한 "가족관계증명서+소득금액증명원+재산관련서류"를 하나의 묶음으로 신청하는 식이다. 즉 **개별 증명서 249종을 낱개로 신청하는 게 아니라, Hondi가 실제로 필요한 조합을 미리 설계해서 "묶음" 단위로 신청**해야 한다.

### 2단계: 현장실사

1단계 승인 후 "현장실사 예정" 통보를 받으면, 제출한 이용 환경 조사표를 근거로 실사가 진행된다. **보안저장소를 자체 구축하기로 했다면 별도 공문으로 실사를 추가 요청**해야 한다.

### 3단계: 보안저장소 구축 (자체구축 시)

보안저장소 규격·체크리스트를 준수해야 한다(「본인에 관한 행정정보의 제공 등에 관한 고시」 제27조 및 별표6 근거). 자체구축 대신 이미 인증된 외부 보안저장소 서비스를 이용하는 선택지도 있는지는 확인 필요.

### 4단계: 시스템 연계 및 테스트

개발서버 연계 → 샘플데이터 테스트 → 운영서버 개통 순서로 진행된다.

### 5단계: 운영

승인 후에도 지속 의무가 따른다 — 연계 중단 시 30일 이내 사전고지 의무, 본인정보 제공 내역 정기 점검 등.

---

## 3. 이용기관의 상시 의무 (등록 후에도 계속됨)

- 전담관리자 지정·유지
- 본인정보 송수신 시 안전한 연계(보안) 확보
- 사용자 본인확인을 위한 **신뢰할 수 있는 인증수단** 자체 보유
- 보안저장소 유지·관리
- 이용 내역 저장·정기 점검

이건 "API 키 하나 발급받으면 끝"이 아니라 **지속적인 보안·운영 체계를 갖춘 기관**이어야 한다는 뜻이다.

---

## 4. 민간기업도 가능한가 — 그렇다, 실사례 확인됨

성장 이력을 보면 민간기업 참여가 이미 정착돼 있다:

| 시점 | 사례 |
|---|---|
| 2020-10 | 공공 마이데이터 서비스 활용기관 1차 업무협약(16개 기관) |
| 2022-06 | 금융기관 수신·여신업무 적용 개시 |
| 2023-12 | **보험·증권·통신분야** 활용기관 업무협약(8개 기관) |
| 2026-01 | **기업 공공 마이데이터** 업무협약식(10개 기관, 신설 트랙) |

특히 2026-01 신설된 "**기업 공공 마이데이터**"는 개인이 아니라 **사업자(기업)의 행정정보**(사업자등록번호 등)를 다루는 트랙이다 — Hondi의 K-Market 소상공인 입점·위치기반사업자 등록 같은 절차와 직결될 가능성이 있다. "기업문서보관함"이라는 용어도 확인됐다.

---

## 5. 정직한 평가 — Hondi(1인 스타트업)에게 이건 무거운 과제다

이번 조사로 알게 된 것 중 가장 중요한 건 **이게 "신청하면 자동승인"되는 data.go.kr 오픈API와는 완전히 다른 급의 절차**라는 점이다.

| 항목 | data.go.kr 오픈API | 공공 마이데이터 이용기관 |
|---|---|---|
| 승인 방식 | 자동승인 | 심사 + 현장실사 |
| 필요 서류 | 없음(신청서 작성만) | 5종 공문 |
| 기술 요건 | serviceKey 발급 | 보안저장소 구축·유지, 전담관리자 지정 |
| 소요 기간(추정) | 즉시 | 수주~수개월(★확인 필요★) |
| 지속 의무 | 없음 | 정기 점검·보고 의무 |

**제주 필드테스트 단계에서 이 경로를 정식으로 추진하는 건 시기상조일 수 있다.** 보안저장소 구축·심사만으로도 상당한 개발 리소스가 필요하다.

## 6. 대안 — 더 가벼운 경로도 있다

`user_guide.pdf`에서 확인한 "**간편보내기**" 방식은 훨씬 가볍다:

1. Hondi가 QR코드를 사용자에게 제시
2. 사용자가 정부24 앱으로 QR을 스캔 → 로그인
3. 사용자가 "보내고 싶은 정보"를 직접 선택해 Hondi로 전송

이 방식은 **Hondi가 이용기관으로 정식 등록되지 않아도** 사용자가 매번 능동적으로 정보를 보내는 형태로 작동할 가능성이 있다(★확인 필요 — QR 수신측도 이용기관 등록이 전제조건인지, 아니면 더 낮은 문턱의 참여 방식이 따로 있는지★). 만약 등록 없이도 가능하다면, 제주 필드테스트 단계의 현실적 대안이 될 수 있다.

## 7. 결론 — 단계적 접근 제안

| 단계 | 방식 | 시점 |
|---|---|---|
| 1단계(현재) | 사용자가 직접 정부24/읍면동에서 발급 → Hondi는 안내만(현재 SP들의 §5 "대안 채널") | 즉시 가능 |
| 2단계 | "간편보내기"(QR) 방식 검증 — 등록 없이 되는지 확인 | 제주 필드테스트 중 검토 |
| 3단계 | 정식 "이용기관" 등록 추진 — 묶음정보를 Phase 1 25건 기준으로 설계 | 전국 확장 이후, 보안저장소 구축 리소스 확보 후 |
| 3단계-부가 | "기업 공공 마이데이터"(2026-01 신설) 별도 트랙 — K-Market 입점 소상공인 대상 | 3단계와 병행 검토 |

## 8. 미해결 — 다음에 확인할 것

1. **간편보내기(QR) 방식이 이용기관 등록 없이 가능한지** — 가장 시급한 확인 사항(★§6★)
2. 이용기관 승인까지 실제 소요 기간(추정치 없음)
3. 보안저장소 자체구축 대신 쓸 수 있는 인증된 외부 서비스가 있는지
4. Phase 1 25건이 공공 마이데이터의 249종 목록에 실제로 포함되는지 대조(아직 안 함)
5. "기업 공공 마이데이터"(2026-01 신설)의 구체적 요건 — 개인용과 별도 절차인지
