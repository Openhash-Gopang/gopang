# SP-부서 입출력 마스터 목록 (중앙정부 + 제주 지방정부)

> 작성일: 2026-07-09
> 목적: 각 부서를 "input → output" 함수로 정의해, 이후 개별 `SP-{부서명칭}` 문서(해당 부서 업무를 보조하는 SP)를 작성하기 위한 밑그림
> 방법론: "위치기반사업자 등록" 사례에서 확인한 기관=함수 관점(병원: 환자→치료, 사업자: 원자재→상품)을 전체 정부조직에 적용
> 상태: 초안 — 각 부서의 input/output은 **핵심 기능 1~2개로 압축**한 것이며, 실제 SP 작성 시 세분화 필요

---

## 0. 표기 규칙

| 열 | 의미 |
|---|---|
| SP 문서명 | 향후 작성할 개별 SP 파일명 (`SP-{부서명}`) |
| Branch | org_profiles.branch 분류 (legislative/judicial/admin_central/admin_local/public_institution) |
| Input | 이 부서가 받아들이는 것(민원·신청·자원·정보) |
| Output | 이 부서가 산출하는 것(허가·처분·서비스·재화) |

## 7. 미확정 · 검증 필요 항목

1. **공소청**: 2026-10-01~02 검찰청 폐지·공소청/중수청 신설이 예정돼 있으나 아직 시행 전(오늘 기준 검찰청이 현행). 시행일 도래 시 이 문서와 실제 SP를 함께 갱신해야 함.
2. **제주 자치경찰**: 제주도 실국 목록에서 자치경찰 조직이 별도로 확인되지 않음 — 별도 확인 필요.
3. **각 부서 input/output은 핵심 기능 1~2개로 압축한 것**이라, 실제 개별 `SP-{부서명}` 작성 시에는 하위 실·국·과 단위로 더 세분화된 input/output이 필요할 가능성이 큼(예: 국세청 하나에도 소득세·법인세·부가세·사업자등록 등 여러 개의 독립적 input/output 쌍이 존재).
4. **재외동포청·우주항공청 등 신설 기관**은 아직 실무 절차가 초기 단계라 input/output이 상대적으로 추정에 의존함.
