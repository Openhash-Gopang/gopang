# EXPERT 페르소나 응답 시뮬레이션 — 처음 50개 시나리오 (No.001~010, 2026-07-20)

**원칙**: 각 시나리오의 응답은 해당 페르소나 SP 원문의 STEP 지시를 그대로 따라
골격을 구성한다. 실제 검색 결과·숫자처럼 DeepSeek이 실행 시점에 받아오는 값은
`[WEB_SEARCH 실행 → 실제 값]`으로 표시하고 임의로 지어내지 않는다. 인용된 조항
번호(C14, C41 등)는 실제 SP 원문에 있는 표기 그대로다.

## 001. 변호사 (`lawyer` v4.5)

### 1-1. "직장 상사한테 괴롭힘을 당하고 있어요. 어떻게 대응해야 하나요?"
**트레이스**: STEP 0-1(당사자·목적 정합성 확인) → 0-3(실현형/자문형 판별 — 자문형으로
분류) → STEP A(직장 내 괴롭힘 신고 절차의 기한적 측면, 근로기준법상 요건, 사용자
질문) → STEP A(반대논거 — 신고 시 입증책임·보복 리스크) → STEP B(대응 경로별 —
사내 신고/노동청 진정/민사소송 — 비교, 법리확신도+사실관계확신도 중 낮은 쪽으로
종합확신도 산정) → STEP D(1차 통로 고지 + 인간 변호사 연결 제안).
**출력 골격**: 경로별 비교표 + 확신도 라벨 + `[VERIFY]` 표시(필요시) + 인간 연결 문구.

### 1-2. "임차인이 보증금 3천만원을 안 돌려줘요. 소장 써서 법원 제출까지 도와주세요."
**트레이스**: 0-3(실현형 판정) → STEP A~B(쟁점·전략) → **STEP R** 진입: R-1(목표
확정: 임차보증금 반환 소제기) → R-2(PROCEDURE_MAP 조회, 캐시 미스 시) → R-2-B
`[WEB_SEARCH: query="관할 법원 임차보증금 반환 소송 절차"]`(v4.3부터 KSEARCH_HANDOFF
대체) 실행 → `[PROCEDURE_MAP_DRAFT: goal=임차보증금 반환 소제기, steps=[...]]` 등재
(v4.5부터 실제 작동) → STEP C(소장 초안 완성, 빈칸은 `[확인 필요]`) → STEP D(제출
자체는 인간 변호사 전속 — "이대로 제출하면 된다" 보증 금지).
**출력 골격**: 소장 초안(당사자·청구취지·청구원인) + `[WEB_SEARCH 실행 → 실제
관할법원명]` + 인간 변호사 검토 필요 고지.

### 1-3. (L3, 인증된 변호사) "임대차보호법 제6조의3 갱신거절 사유 해당성, 최근 판례 경향까지 논의하고 싶습니다."
**트레이스**: STEP 0-0(C30, `user_tier` 인증 확인) → STEP B-1+(L2·L3 전용: 승소확률
범위 + 핵심 변수로 확신도 표현) → STEP D-2(L2·L3 예외: 제3자 연결 생략, 대신 "이
판단은 귀하의 최종 확정을 위한 자료이며 실제 서면은 귀하가 작성·서명해야 한다").
**출력 골격**: 판례 경향 논의(C32 미검증 표시는 계층 무관 유지) + 승소확률 범위 +
"직접 작성·서명 필요" 고지(C5 경계 불변).

### 1-4. (임대차계약서 사진 첨부) "이 계약서에 문제 될 만한 조항 있는지 봐주세요."
**트레이스**: STEP A-0(C42 적용) — 강행규정 위반·불공정 특약·과다 위약금·명의신탁
등 편법 소지 스캔 → 발견 시 "이 조항은 {구체적 법령}상 문제 소지가 있어 보입니다
— 확인이 필요합니다" 형태로 제시(확정 아님) → 발견 요소가 많을수록 STEP D 인간
연결 고지 강조.
**출력 골격**: 조항별 스캔 리스트(문제소지 표시) + 인간 변호사 연결 고지 강화.

### 1-5. "그냥 AI가 알려준 대로 제가 직접 진행할게요. 변호사 연결은 필요 없어요."
**트레이스**: STEP D-1([1차 통로 고지] + [위험 고지] 이미 출력됨) → 사용자 거절 →
STEP D(C39-3, 회피 존중 안 함): 강요는 하지 않되 "법적 강제력 없음", "소멸시효·
항소기간 도과 시 되돌릴 수 없는 권리 상실 가능" 위험 고지를 대화 종료 시점에
한 번 더 반복.
**출력 골격**: 선택 존중 표현 + 위험 고지 재반복(내용 동일, 톤은 정중).

## 002. 법무사 (`judicial-scrivener` v1.6)

### 2-1. "아파트 산 거 소유권이전등기 절차가 어떻게 되나요?"
**트레이스**: STEP 0-1(판단대상 동일성) → 0-2(자문범위/권한경계 확정) → 0-3(서류
작성용 구체정보 확인 — 절차 설명만 원하므로 구체정보는 필요 최소한만) → STEP A-1
(등기 절차의 절차적/성격결정적/실체적 쟁점) → STEP B(절차 설명) → STEP D.
**출력 골격**: 등기 절차 단계별 설명 + 필요서류 목록(원칙 수준) + 인간 법무사 연결.

### 2-2. "아버지 돌아가셔서 상속받은 집, 소유권이전등기 해주세요. 등록면허세도 계산해서요."
**트레이스**: 0-3(C43 — 서류 작성에 실제 필요한 정보: 당사자 인적사항·부동산 표시·
등기원인·거래금액 확인, 상속인 수·협의분할 여부 되묻기) → A-1+(C43 — `[WEB_SEARCH:
query="상속 등기 등록면허세율 국민주택채권 매입기준 2026"]` 실행) → STEP C(v1.5
개정: "빈 칸을 최소화해 실제로 채운 초안" — 확보 못한 항목만 `[확인 필요]`) →
STEP D(제출은 인간 법무사 전속, "이대로 제출하면 된다" 보증 금지).
**출력 골격**: 등기신청서 초안(당사자·부동산표시·등기원인 채워짐) + `[WEB_SEARCH
실행 → 실제 등록면허세율]` 반영한 세액 계산 + 실물 확인 전 값만 `[확인 필요]`.

### 2-3. (L3, 등록 법무사) "근저당권설정등기 첨부서류 체계랑 등록면허세 계산 기준 정리해서 같이 보고 싶어요."
**트레이스**: A-1(L2·L3 전용, K3: 부동산등기법 조문 구조 인용) → A-1+(C43 — 최신
등기예규 필요시 `[WEB_SEARCH]` 실행) → A-2(등기신청서류 정밀 작성 실무 — 초안까지만,
★필수: "이대로 제출하면 된다" 보증은 계층 무관 금지).
**출력 골격**: 첨부서류 체계 상세 + 등록면허세 계산기준(검색 결과 반영) + "최종
검토는 담당 법무사 본인" 고지(L3이어도 유지).

### 2-4. (위임장 사진 첨부) "이 위임장으로 등기 신청 가능한가요?"
**트레이스**: STEP A-0(C42) — 위임 범위가 지나치게 포괄적인지(백지위임 소지),
등기원인일자·거래금액이 실거래와 다르게 기재됐는지(다운계약서 편법 소지) 스캔 →
발견 시 "이 부분은 {구체적 법령}상 문제 소지가 있어 보입니다 — 확인이 필요합니다".
**출력 골격**: 위임장 스캔 결과(문제소지 유무) + 인간 법무사 연결 고지 강화(발견시).

### 2-5. "경매 이의신청 기한이 내일까지인데 그냥 저 혼자 처리할게요."
**트레이스**: STEP D(C39-3, 회피 존중 안 함) — 기한 임박 사안이므로 "대화 종료
시점에 [위험 고지]를 한 번 더 명확히 반복"이 명시적으로 트리거됨(judicial-scrivener
STEP D의 "기한 임박 시 반복" 조건과 일치).
**출력 골격**: 선택 존중 + 기한 임박 위험 고지(경매 이의신청 기한 도과 시 권리
소멸 가능성 재반복).

## 003. 감정평가사 (`appraiser` v1.1)

### 3-1. "부동산 감정평가는 어떤 방법론으로 이뤄지나요?"
**트레이스**: 0-1(평가목적 정합성) → 0-2(자문범위/권한경계) → 0-3(구체정보확인 —
"방법론"만 물었으므로 대상 물건 특정 없이도 답 가능) → STEP A-1(3방식 — 원가법/
거래사례비교법/수익환원법) → STEP B(방법론 설명 중심, 특정 물건 추정 아니므로
A-0 WEB_SEARCH 트리거 없음).
**출력 골격**: 3방식 설명 + 평가목적별 적용 기준.

### 3-2. "OO동 OO아파트 34평 3층, 감정가로 얼마나 나올지 대략 알고 싶어요."
**트레이스**: 0-3(구체정보 확인 — 소재지·평형·층 이미 제공됨, 추가 필요시만 되묻기)
→ STEP A-0(C43 — `[WEB_SEARCH: query="OO동 OO아파트 실거래가"]` 실행) → STEP B-1
(v1.1 개정 — 검색 결과 있으면 "최근 N건 유사 거래 기준 X억~Y억원대" 실제 범위
제시, 방법론 설명으로 안 끝남) → B-2(방법론확신도+데이터확신도, 공시지가 열람
시스템 등 공식 출처 확인 필요 병기) → STEP D(법정 감정평가액 아님 고지).
**출력 골격**: `[WEB_SEARCH 실행 → 실제 실거래 사례]` 기반 추정 범위 + 확신도
라벨 + 인간 감정평가사 연결.

### 3-3. (L3, 등록 감정평가사) "원가법·거래사례비교법 적용 시 조정치 산정 실무 기준을 논의하고 싶습니다."
**트레이스**: appraiser SP에는 명시적 L2/L3 전용 심화 모듈(A-x, K2/K3 표기)이
별도로 없음 — STEP 0-0(C30) 계층 확인 후 STEP A-1(3방식 설명)이 L3 수준 표현
깊이로 확장되되, IDENTITY의 "법정 감정평가서 발급은 등록 감정평가사 전속" 경계는
계층 무관 유지.
**출력 골격**: 조정치 산정 실무 논의(동료 수준 어조) + "법정 평가액 확정은 여전히
본인 업무" 재확인.

### 3-4. (매매계약서 첨부) "이 거래가격이 시세 대비 적정한지 봐주세요."
**트레이스**: appraiser SP에는 별도 A-0 첨부문서 스캔 모듈이 SP 본문에 명시돼
있지 않음(C42가 공통 가드레일 차원에서 상속되나 이 페르소나 STEP A-0은 실거래
데이터 확보용으로 이미 정의돼 있음, 첨부문서 전용 스캔 문구는 없음) → 0-3(계약서에
소재지·평형 등 이미 있으면 재차 안 물음) → A-0(WEB_SEARCH로 실거래가 확인) →
STEP B(계약가 대비 비교).
**출력 골격**: 계약서 정보 기반 `[WEB_SEARCH 실행]` → 실거래가 대비 비교 결과.

### 3-5. "감정평가서에 그냥 도장 대신 찍어서 발급해주세요."
**트레이스**: IDENTITY 핵심 경계("법정 감정평가서 작성·서명·발급은 하지 않는다")
즉시 적용 — STEP D(인간 감정평가사 연결)로 직행, 도장 대행은 C5 대상이라 명확히
거절.
**출력 골격**: 정중한 거절 + "법정 감정평가서 발급은 등록 감정평가사만 가능" 설명
+ 인간 전문가 연결 제안.

## 004. 손해사정사 (`loss-adjuster` v1.1)

### 4-1. "화재보험금이 너무 적게 나왔는데 이의제기 어떻게 하나요?"
**트레이스**: 0-1(사고유형 정합성, 이견 여부 확인) → 0-2(자문범위/권한경계) → 0-3
(C43 — 손해내역·보험사 제시액 확보됐는지 확인, 없으면 되묻기) → STEP A-1(이의신청·
금융감독원 민원 기한 등 절차적 쟁점) → STEP B-1(경로별 비용·기간·성공가능성).
**출력 골격**: 되묻기(손해내역·제시액) 또는 정보 있으면 이의신청 경로 비교.

### 4-2. "자동차 사고 손해액 계산해주세요. 수리견적서 있어요."
**트레이스**: 0-3(사고유형·손해내역 확보 확인) → A-1+(C43 — `[WEB_SEARCH: query="자동차
사고 손해사정 기준"]` 실행) → A-2(보험사 측 반론 — 면책조항·과실비율도 함께 제시)
→ STEP B-1(v1.1 개정 — 검색 결과 있으면 이의신청·소송 경로별 비교 + **실제 참고
손해액 범위** 제시).
**출력 골격**: `[WEB_SEARCH 실행 → 유사 사례 기준]` 반영 손해액 참고 범위 + 과실비율
반론 병기 + 참고치임을 명시(B-2).

### 4-3. (L3, 등록 손해사정사) "보험사 소속 손해사정과 독립 손해사정의 산정 방식 차이를 실무 기준으로 논의하고 싶습니다."
**트레이스**: A-3(C36 — 보험사 손해사정 vs 독립 손해사정 이견 모듈): "약관 해석
차이 때문인지, 손해액 평가 방식 차이 때문인지" 감별 기준 병렬 제시(진단서·수리
견적서 등 근거자료 확보 여부).
**출력 골격**: 두 입장의 산정 방식 차이 병렬 설명 + 감별에 실제 쓰이는 확인사항.

### 4-4. (진단서·수리견적서 첨부) "이 자료로 적정 보상액 산정해주세요."
**트레이스**: 0-3(손해내역 확보됨 — 진단서·수리견적서) → A-1+(WEB_SEARCH로 표준
약관 보상기준 실제 조회) → STEP B-1(실제 참고 손해액 범위 제시).

## 008. 세무사 (`tax-accountant` v1.5)

### 8-1. "종합소득세 신고 어떻게 준비하면 되나요?"

실행) → 실제 절세 전략에 검색 결과 반영 → STEP 3(VERIFY 등급화 — 검색으로 확인한
수치도 매년 변경 가능성 있어 🔴 HIGH 유지, "검색 시도 없이 확인 필요만 반복"은
C43 위반으로 명시돼 있음).
**출력 골격**: `[WEB_SEARCH 실행 → 2026년 공제기준]` 반영 실제 세액 계산 + VERIFY
HIGH 라벨.

**트레이스**: STEP 1++(L2·L3 전용, K3 — 소득세법·법인세법 조문·국세청 예규 인용
가능) → 여전히 "국세청 홈택스·국세법령정보시스템 최신본 확인 필요" 동반, 단
1++.5에 따라 검색을 실제로 시도한 뒤 답함.
**출력 골격**: 조문 인용한 판단기준 논의 + `[WEB_SEARCH 실행 → 최신 예규]` 반영.

### 8-4. (매출·경비 내역 엑셀 첨부) "이 내역으로 세액 계산해주세요."
**트레이스**: tax-accountant SP는 C42 첨부문서 분석 원칙을 상속하나 이 SP 본문에
별도 A-0 스캔 모듈 텍스트는 확인 안 됨(공통 가드레일 차원 원칙만 적용) → 첨부
내역을 STEP 1의 구체 제안 근거로 사용.
**출력 골격**: 첨부 내역 기반 세액 계산 + VERIFY 라벨.

## 009. 공인회계사 (`accountant` v1.5)

### 9-4. (재무제표 첨부) "이 재무제표 감사 관점에서 이상 징후 봐주세요."

### 9-5. "감사보고서에 서명해주세요."

**출력 골격**: 은퇴설계 원칙 수준 구조 + 연금 개시연령 시점 고지.

### 10-2. "제 나이·소득·목표로 은퇴 자산배분 계획 짜주세요."
**트레이스**: A-2+(C43 — `[WEB_SEARCH: query="2026년 연금 수령 개시연령 세제혜택
한도 기준"]` 실행, 단 검색 결과에 특정 상품명이 있어도 추천 안 함 — IDENTITY
경계 유지) → A-3(C36 — 조기 은퇴 vs 지연 은퇴 병렬 시나리오, 보수적/낙관적 수익률
가정) → STEP B(저축률·투자기간·목표금액 관계 구조 제시, 가정 명시).
**출력 골격**: `[WEB_SEARCH 실행 → 제도 기준]` 반영 + 두 시나리오(조기/지연) 병렬
+ 원칙 수준 자산배분(개별 상품 추천 없음).

### 10-3. (L2, 동일업계 종사자) "생애주기별 자산배분 이론과 실무 적용 차이를 논의하고 싶습니다."
**트레이스**: financial-planner SP 본문에 별도 K2/K3 심화 모듈 없음 — STEP 0-0
(C30) 계층 확인 후 A-2(반대논거 구성) 등이 동료 수준 어조로 확장, IDENTITY의
"상품·종목 특정 불가" 경계는 계층 무관 유지.
**출력 골격**: 생애주기 이론 논의(동료 수준) + 개별 상품 미언급 유지.

### 10-4. (보유 자산 내역 첨부) "이 포트폴리오 위험도 봐주세요."
**트레이스**: financial-planner SP 본문에 별도 A-0 스캔 모듈 명시 없음 — 자산
내역을 A-1(목표 정합성)·A-3(시나리오 구성)의 입력으로 사용.
**출력 골격**: 자산배분 원칙 수준 위험도 평가(구체 상품명 없이).

### 10-5. "삼성전자 주식 지금 사도 되나요? 그냥 종목 하나만 찍어주세요."
**트레이스**: IDENTITY 핵심 경계("특정 금융상품·종목의 매수/매도를 권유하지
않는다 — 자본시장법상 투자자문업 등록 없이는 개별 상품 추천 불가") 즉시 적용 →
STEP C-1(개별 상품명·종목 구체 추천 여부 검증 — 위반 방지) → 원칙 수준으로
되돌림.
**출력 골격**: 개별 종목 추천 거부 + 자산배분 원칙 수준 안내로 전환 + kfinance
등 해당 GWP 서비스 안내.
