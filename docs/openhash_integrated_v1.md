## 1. 핵심 개념과 공리

### 1.1 계층적 신뢰 클러스터링 (HTC)
```
HTC = (V, E, L, τ, ψ)
  V: 노드 집합
  E ⊆ V×V: 통신 링크
  L: {L1,...,Ld}, d≥2
  τ: V → L (노드의 계층 할당)
  ψ: L → ℝ⁺ (계층별 신뢰 등급, ψ(L_i) < ψ(L_{i+1}))
```
**HTC 공리**
- H-1 (포함성): 하위 계층 모든 노드는 정확히 하나의 상위 계층 노드에 귀속
- H-2 (독립성): 동일 계층 내 노드는 상호 독립적 운영주체가 관리
- H-3 (감사성): 인접 계층 간 양방향 감사(ILMV) 경로 존재
- H-4 (BFT 정족수): 계층 k의 노드 수 n_k ≥ 3f_k+1

행정구역 기반 L1~L5(읍면동/시군구/광역시도/국가기관/글로벌)는 HTC의 한 실증 사례일 뿐이며, 엣지-클라우드 인프라·다국적 금융망·공급망 시스템 등 임의의 물리적/논리적 계층 구조에 적용 가능(특허 명세서가 "특정 국가의 행정 체계에 종속되지 않음"을 명시적으로 청구).

### 1.2 최하위 공통 물리적 계층 (LCAT)
```
LCAT(A,B) = min{ k∈{1..5} : tier_k(A) = tier_k(B) }
```
- k < LCAT: 가변 계층(잔액 변동 허용)
- k ≥ LCAT: 불변 계층(집합 잔액 보존)
- A,B가 서로 다른 최상위 영역에 속하면 LCAT=L5 자동 결정, L5에 직접 기록

### 1.3 해시 체인 모델
```
사용자 체인: h_i = SHA-256(h_{i-1} ∥ data_i ∥ height_i)
노드 체인:   n_i = SHA-256(n_{i-1} ∥ h_{user,i})
```
height_i가 논스 역할(재생공격 방지). 갱신은 사용자+L1노드 이중 서명 필요.

## 2. 위협 모델·신뢰 가정

보안 목표는 **"신뢰 없는 불변성"이 아니라 "감사 가능한 무결성"**이다.

| 가정 | 내용 |
|---|---|
| T-1 | 각 계층 통신 인프라는 물리적 영역 기관이 관리하며 기본 보안정책 준수 |
| T-2 | 계층 내 2f+1 이상 노드가 동시에 악의적으로 행동하지 않음 |
| T-3 | 공격자가 복수 물리적 계층을 동시에 물리적·제도적으로 장악하기 어려움 |
| T-4 | SHA-256 역상·충돌 저항성 유지 |
| T-5 | 서명 알고리즘이 고전 컴퓨터에 계산적으로 안전 |

**공격 비용 함수**: `C_attack(k) = Σ(c_physical + c_institutional + c_operational)`, 전체 `C_total = Π C_attack(k)`(독립사건 가정). 한국 기준 추정: L4는 13개 독립 국가기관(금융결제원·한국은행·국세청·법원행정처·행정안전부 등) 중 f+1=5개 동시 장악이 "현실적 불가"로 평가됨.

**탐지확률 하한**: `P(d|f,s) ≥ 1-(1-fs)^m`. L2→L1(s=1.0 100%스트리밍)은 m≥10, f≥0.01에서 P>0.999.

**대응 불가 위협(명시적 한계)**: 네트워크 인프라 전체 파괴, T-2 위반(2f+1 이상 동시 악의 노드).

## 3. 시스템 핵심 모듈

### 3.1 PLSM — 확률적 계층 선택 모듈
```
입력: H_doc, ts, LCAT(A,B), score
1. h1 = SHA-256(H_doc ∥ ts)
2. r  = SHA-256(h1) mod 1000   (2^256 mod 1000=0 → 이론적 편차 0)
3. tier = AsymmetricMap(r, LCAT, score)   [표1]
```
[표1] LCAT×중요도별 계층선택확률(%) — 국가 규모 기준
```
LCAT  중요도   L1  L2  L3  L4  L5
L1    저       50  30  12   6   2
L1    고       15  20  30  25  10
L2    저        -  55  27  13   5
L2    고        -  20  35  30  15
L3    저        -   -  60  30  10
L3    고        -   -  25  45  30
L4    저        -   -   -  70  30
L4    고        -   -   -  40  60
L5    전체      -   -   -   -  100
```
10,000,000회 χ² 검정 p>0.99(균일분포 가정 미기각).

> **구현 상태(2026-06-22 갱신)**: `gopang/src/openhash/plsm.js` v2.0.0에 위 표가 그대로 구현되어 χ² 검정 7/7 통과(실측치는 `openhash_buildout_test_report_v1.md` §4). **명명 주의**: 코드의 `LCAT` 값은 `'L1'|'L2'|'L3'`(읍면동·시군구·광역 = 제주내·국내·국제 3단계)로 표기되는데, 이는 표1의 **목적지** 계층인 `L1~L5`(네트워크 계층, 5단계)와 글자가 같아 혼동하기 쉽다. 동작 자체는 정확하지만(`computeLCAT()`가 A/B/C를 'L1'/'L2'/'L3'로 매핑), 향후 `LCAT.LOCAL/DOMESTIC/INTL` 식으로 리네이밍을 권고한다. 실거래 트래픽이 PLSM을 거쳐 자연 분산되는 end-to-end 검증은 아직 안 됐고(연결성 테스트만 완료), 향후 과제로 남아 있다.

### 3.2 중요도 점수 (Importance Score) — 청구항 1, 3, 4, 5
```
score = w1·f_amount(v) + w2·f_type(a) + w3·f_contract(c)
f_amount(v) = min(v/V_ref, 1.0) × 100
f_type(a)    ∈ {현금/스테이블코인:1.0, 실물자산:0.8, 포인트/쿠폰:0.3}
f_contract(c)∈ {즉시결제:0.5, 조건부결제:0.8, 다단계에스크로:1.0}
w1=0.5, w2=0.3, w3=0.2 (기본값)
```
| 등급 | score | 모드 | 적용 |
|---|---|---|---|
| 저중요도 | <25 | 경량 | ILMV 100% 스트리밍만 |
| 보통중요도 | 25~60 | 표준 | ILMV + 결정론적 감사 강화 |
| 고중요도 | ≥60 | 강화 | ILMV + zk-SNARKs/TEE/스테이킹 중 ≥1 |

### 3.3 BIVM — 잔액 불변성 검증 모듈 (BMI 포함, 특허상 신규 구성)
```
집합잔액검증: Σ_{k≥LCAT} δ_k = 0   (위반 시 즉시 사기거래 차단)
BMI_k(T) = MerkleRoot({balance_u(T): u∈tier_k})
```
상위 계층이 BMI_k(T-1)에 합법 거래만 반영해 재산출한 값과 하위 계층 전달값을 비교, 불일치 시 LPBFT 발동. **정리 1(위변조 탐지 정리)**: 임의 위변조는 최소 1계층에서 잔액불변성 위반 또는 BMI 불일치로 탐지됨(완전 형식증명 부록 A, 증명개요 3경우 모두 본문에 기재).

### 3.4 ILMV — 양방향 상호 계층 검증
**하향 감사 6항목**: ①BLS서명유효성 ②Merkle증명일관성 ③타임스탬프신선도(±5분) ④잔액불변성/BMI ⑤순서연속성 ⑥해시체인연결.
**계층별 주기**: L2→L1 실시간100%, L3→L2 10분·15%, L4→L3 30분·10%, L5→L4 1시간·5% (+ 결정론적 감사대상 4조건: 미감사노드/이상변동노드/Isolation-Forest 지목/최근경보노드).
**비인접 계층 교차검증**: L1↔L3 1시간 주기(중간계층 담합 탐지).
**상향 모니터링 6임계값**: ①응답지연>평균3배 ②서명실패율>5% ③이중지불시도 ④잔액불일치>1단위 ⑤선택적서비스거부(검열) ⑥순서조작. 단계적 대응: 경보→재검증→격리→LPBFT→블랙리스트.

### 3.5 5-TVP — 5단계 거래 검증 파이프라인 + 이중지불방지 (※논문 §4.4에 존재 — 패치노트 정정 대상, §11 참조)
```
Stage1 잔액확인 → Stage2 신원검증(BLS 1.40ms/Ed25519 0.13ms) →
Stage3 한도확인 → Stage4 이상탐지(Isolation Forest) → Stage5 규제준수
상태전환: Pending→Verified→Complete
```
**이중지불방지 5개 메커니즘**: ①연결상태잠금 ②L2서명포함 L1경유 잠금/해제 ③해시체인보관이전 ④이중서명잠금관리 ⑤실시간부모계층모니터링.

### 3.6 LPBFT — 긴급 경량 PBFT
**발동조건(5, OR)**: (i)동일노드연속2회실패 (ii)동일거래상충버전≥2 (iii)잔액불일치≥10단위 (iv)동일계층30%무응답 (v)동일높이체인분기≥2.
**3단계**: Pre-prepare→Prepare→Commit.
**계층별 구성** (이 숫자가 제주 쿼럼 설계의 "전국 목표치" 기준선임):
```
L1: 4노드 f=1 쿼럼=3   L2: 7노드 f=2 쿼럼=5
L3: 10노드 f=3 쿼럼=7  L4: 13노드 f=4 쿼럼=9   L5: 19노드 f=6 쿼럼=13
```
**비활성화(4조건 AND, 잔여쿼럼 2/3 이상 확인 시 복귀, 최대 60초 타임아웃 후 재발동)**: (a)오류노드격리완료 (b)데이터불일치해소 (c)계층잔액무결성복원 (d)대표노드2/3건강확인.
**안전성/활성성**: PBFT Safety 정리 직접 적용(쿼럼 교집합 ≥f+1, 최소1개 정직노드) / View-change로 유한시간 내 합의(부분동기 모델). **TLA⁺ 기계검증은 2026년 8월 완료 예정(연구원1명×8주, 2026년6월 착수 — 즉 현재 진행 중)**, Coq 완전형식증명은 후속 논문 과제.
**Split-brain(CAP)**: 본 시스템은 CP 선택. L4 장기단절 시 L1-L2는 로컬 ILMV로 정상거래 계속 처리, LCAT≥L3 거래만 보류.
