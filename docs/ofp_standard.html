Open Federation Protocol

경쟁 플랫폼 간 상호운용 표준 — OFP v1.0

경쟁자는 적이 아니라, 더 나은 생태계의 공동 건설자입니다.
OFP는 서로 다른 플랫폼의 사용자가 검색하고, 채팅하고, 결제하고, AI 서비스를 공유할 수 있도록 공통 언어를 정의합니다.

Design Principles

6대 핵심 원칙

OFP는 고팡의 경쟁 우위를 보호하기 위한 장벽이 아닙니다. 더 나은 AI 시민 서비스 생태계를 만들기 위한 공통 기반입니다.

🔓
개방성 Openness
모든 사양은 GitHub에 공개되며 누구나 읽고 구현할 수 있습니다.
⚖️
중립성 Neutrality
고팡이 표준을 독점하지 않습니다. 참여 플랫폼이 거버넌스에 동등하게 참여합니다.
🕸️
분산성 Decentralization
중앙 서버 없이 플랫폼 간 직접 통신합니다. P2P Federation 구조입니다.
🔐
프라이버시 우선
사용자 데이터는 각 플랫폼의 PDV에 남습니다. OFP는 메시지만 중계합니다.
🧩
점진적 채택
전체 스펙을 구현하지 않아도 부분 참여가 가능합니다. 모듈형 설계입니다.
🏆
경쟁 보호
연결하되 흡수하지 않습니다. 플랫폼 간 사용자 이탈을 강제하지 않습니다.
Protocol Modules

6개 핵심 모듈

필수 2개만 구현해도 OFP에 참여할 수 있습니다. 나머지는 역량에 따라 순차 채택합니다.

필수
사용자 탐색
OFP-DISCOVER
플랫폼 간 사용자 검색 및 프로필 조회. WebFinger RFC 7033 기반. @user@platform.com 형식의 연합 주소 체계를 정의합니다.
필수
메시지 중계
OFP-MSG
크로스 플랫폼 실시간 채팅. Ed25519 서명 + X25519-ChaCha20-Poly1305 E2E 암호화. 텍스트·첨부파일·서비스 결과 공유를 지원합니다.
선택
AI 서비스 브릿지
OFP-BRIDGE
K-Law·K-Health 등 AI 서비스를 타 플랫폼 사용자도 호출 가능. 경쟁 플랫폼이 더 뛰어난 AI를 개발하면 고팡 사용자도 이용할 수 있습니다.
선택
결제 게이트웨이
OFP-PAY
GDC 크로스 플랫폼 결제. Atomic Swap으로 양측 원장 동시 정산. 수수료 0%, 실패 시 자동 롤백 보장합니다.
선택
신원 연합
OFP-ID
PDV 기반 크로스 플랫폼 신원 인증. L0~L3 신뢰 수준별 권한 차등 부여. K-Law 준법 이력을 타 플랫폼에서도 인정받습니다.
선택
거버넌스 연합
OFP-GOV
여러 플랫폼 사용자가 공동 안건에 투표. K-Democracy의 DAWN 철학 확장. 의결 결과는 모든 참여 플랫폼에 자동 반영됩니다.
OFP-DISCOVER

사용자 탐색 — WebFinger 기반

서로 다른 플랫폼의 사용자 주소는 @username@platform-domain 형식으로 표준화됩니다.

WebFinger Request / Response 
# 요청 — 플랫폼 B에서 고팡 사용자 탐색
GET /.well-known/webfinger
    ?resource=acct:kim_cheolsu@gopang.net
    &rel=http://ofp.gopang.net/ns/user

# 응답 JSON
{
  "subject": "acct:kim_cheolsu@gopang.net",
  "links": [
    {
      "rel":  "http://ofp.gopang.net/ns/user",
      "type": "application/ofp+json",
      "href": "https://gopang.net/ofp/users/kim_cheolsu"
    },
    {
      "rel":  "http://ofp.gopang.net/ns/inbox",
      "href": "https://gopang.net/ofp/inbox/kim_cheolsu"
    }
  ]
}
OFP-MSG

크로스 플랫폼 메시지 프로토콜

모든 메시지는 Ed25519 서명과 E2E 암호화를 적용합니다. 수신 플랫폼은 발신자의 공개키로 서명을 검증합니다.

OFP Message Object 
{
  "ofp_version": "1.0",
  "type":        "Create",
  "actor":       "https://gopang.net/ofp/users/kim_cheolsu",
  "to":          ["https://nova-ai.kr/ofp/users/park_yeonghui"],

  "object": {
    "type":      "OFPMessage",
    "content":   "안녕하세요!",
    "published": "2026-06-06T10:30:00Z",
    "encryption": {
      "type":       "X25519-ChaCha20-Poly1305",
      "ciphertext": "base64_encrypted..."
    }
  },

  "signature": {
    "type":           "Ed25519Signature2020",
    "signatureValue": "base64_ed25519_sig..."
  }
}
메시지 유형type 값용도
일반 텍스트OFPMessage기본 채팅 메시지
AI 서비스 결과OFPServiceResultK-Law 판결문 등 공유
GDC 송금 요청OFPPaymentRequest크로스 플랫폼 결제
그룹 초대OFPGroupInvite멀티 플랫폼 그룹 채팅
신원 확인OFPIDVerifyPDV 신원 상호 확인
Comparison

기존 연합 프로토콜과의 차별점

ActivityPub·Matrix와 달리, OFP는 AI 서비스 호출과 GDC 결제를 프로토콜 수준에서 표준화합니다.

비교 항목ActivityPubMatrixOFP
주 용도소셜 미디어 피드실시간 채팅AI 서비스 + 채팅 + 결제
AI 서비스 연동없음없음14개 서비스 연동
결제 프로토콜없음없음GDC Atomic Swap
PDV 신원없음없음IPv6 기반 분산 신원
거버넌스단일 조직재단K-Democracy 온체인 투표
최소 구현 부담높음높음모듈 2개로 시작 가능
Required Endpoints

참여 플랫폼 필수 엔드포인트

OFP 참여를 위해 반드시 구현해야 하는 HTTP 엔드포인트입니다. 필수 모듈(DISCOVER·MSG) 기준입니다.

GET
/.well-known/webfinger
사용자 탐색 — RFC 7033 WebFinger
GET
/.well-known/ofp
플랫폼 메타데이터 및 지원 모듈 목록
GET
/.well-known/ofp-services
제공 AI 서비스 레지스트리 (선택)
POST
/ofp/inbox/{username}
외부 플랫폼으로부터 메시지 수신
GET
/ofp/outbox/{username}
메시지 발신 이력 조회
GET
/ofp/users/{username}
사용자 공개 프로필 및 공개키
