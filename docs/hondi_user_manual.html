제1장

혼디(Hondi) 개요

1.1 혼디란 무엇인가

혼디(Hondi, 구 고팡·Gopang)는 전화번호 하나만으로 가입할 수 있는 글로벌 디지털 문명 네트워크입니다. 개인·기관·사물 모두 혼디 네트워크의 구성원(node)이 될 수 있으며, 각 구성원의 프로필은 곧 그 사람의 웹페이지이자 SNS 프로필, 명함, 상점, 재무제표, 법적 증거 저장소가 됩니다.

혼디의 핵심 설계 원칙은 다음 세 가지입니다.

  • 데이터 주권 — 모든 원본 데이터는 서버가 아닌 사용자의 기기(로컬)에 저장됩니다. 서버에는 위변조 불가능한 해시(지문)만 보관됩니다.
  • 법적 확실성 — K-Law 시스템이 모든 대화와 거래를 실시간 감시하여 불법·편법·위법 행위를 즉시 포착하고 차단합니다.
  • 무국경 경제 — GDC(Global Digital Currency)로 193개국 어디서나 수수료 없이 즉시 결제·환전이 가능합니다.

제2장

가입 및 초기 설정

2.1 가입 절차 전체 흐름

혼디 가입은 전화번호 입력만으로 시작됩니다. 별도의 이메일, 비밀번호, 공인인증서가 필요 없습니다.

  1. 전화번호 입력 — 국가코드 없이 로컬 번호 입력 (예: 01012345678)
  2. E.164 자동 정규화 — 시스템이 국제 표준 형식(+821012345678)으로 변환
  3. GUID 생성 — SHA-256('gopang-phone:' + E.164 번호)를 계산하여 IPv6 형식의 고유 식별자(GUID) 생성. 같은 번호로 재가입해도 동일한 GUID가 부여됩니다.
  4. 통신사 인증 — 해당 국가의 이동통신사 인증을 통해 본인 확인
  5. GDC 지갑 자동 생성 — 가입 기기에서 Ed25519 키페어를 자동 생성. 공개키는 서버에 등록(TOFU), 개인키는 기기에만 보관
  6. 백업 키 확인 (필수) — 개인키를 4단어 시드 형식으로 표시. 이를 확인하고 안전한 곳에 보관해야만 다음 단계로 진행 가능
  7. 닉네임 및 지역 입력 — 동명이인이 있을 경우 handle(@국가코드-번호)로 자동 구분
  8. 이용 약정서 동의 — 미동의 시 가입 완료 불가
  9. 혼디 색상 코드 자동 생성 — GUID를 기반으로 개인 고유의 색상 코드 이미지 생성 및 캐시
  10. LLM 선택 (1주일 무료 체험) — 원하는 AI 언어모델을 선택하여 7일간 무료 사용
  11. 프로필 온보딩 (PA 비서 안내) — 그림자 AI가 프로필 작성을 단계적으로 안내
💡
백업 키(4단어 시드)는 기기 분실 시 계정을 복원하는 유일한 수단입니다. 절대 분실하지 마십시오. 혼디 팀도 이를 복구할 수 없습니다.

2.2 LLM(AI 언어모델) 선택

가입 후 선호하는 AI 언어모델을 선택합니다. 최초 선택 후 1주일간 무료로 제공됩니다.

  • 그림자 AI(개인 비서)는 선택한 LLM을 기반으로 동작합니다.
  • 이후 유료 전환 또는 모델 변경은 설정 메뉴에서 언제든 가능합니다.
  • K-Law, K-Tax 등 전문 서비스는 혼디 자체 모델을 사용하며 별도 선택이 필요 없습니다.

제3장

사용자 프로필

3.1 프로필 = 웹페이지 = SNS = 명함

혼디에서 프로필은 단순한 소개 페이지가 아닙니다. 프로필은 다음 모든 것을 동시에 의미합니다.

  • 개인 웹페이지 — 고유 URL로 외부에서 접근 가능
  • SNS 프로필 — 다른 사용자와 연결, 소통
  • 디지털 명함 — 혼디 색상 코드로 오프라인에서도 공유
  • 상점 페이지 — 사업자의 경우 상품·서비스 목록과 가격 표시
  • 재무제표 허브 — 거래 내역이 자동으로 재무제표에 반영

3.2 프로필 유형

유형대상주요 기능
개인(Individual)자연인일반 프로필, AI 비서, 개인 재무제표
기관(Organization)법인, 단체, 공공기관기관 정보, 소속 직원 연결, 기관 AI 서비스
사물(Thing/IoT)기기, 차량, 장비기기 식별, 자동 보고, 유지관리 이력

3.3 프로필 온보딩 6단계

가입 직후 그림자 AI(PA 비서)가 다음 6단계로 프로필 작성을 안내합니다. 이미 입력된 정보는 재질문하지 않습니다.

  1. 기본 정보 — 이름, 닉네임, 지역
  2. 연락처 및 소셜 — 선택적 연락 채널 등록
  3. 소개 — 자기소개, 직업, 관심사
  4. 사업자 여부 — 사업자라면 사업자 정보, 업종, 사업자등록번호
  5. 상품/서비스 등록 — 사업자의 경우 판매 상품명, 가격, 상세 설명 등록
  6. 공개 범위 설정 — 항목별 공개/비공개/연결된 사용자에게만 공개 설정
📝
프로필 작성은 언제든 중단할 수 있습니다. '나중에 할게요'라고 말하면 즉시 중단되며, 이후 설정 메뉴에서 이어서 작성할 수 있습니다.

3.4 사업자 상품 등록

사업자 프로필을 등록한 경우, 상품과 서비스를 직접 프로필에 올릴 수 있습니다.

  • 상품명, 가격(GDC 또는 자국 통화), 재고, 상세 설명, 이미지를 등록
  • 구매자가 혼디 색상 코드를 스캔하면 바로 해당 상점 프로필로 이동하여 구매 가능
  • 거래 발생 시 재무제표(손익계산서, 대차대조표)가 자동 갱신
  • K-Law가 거래 조건의 합법성을 실시간 검토
제4장

혼디 색상 코드

4.1 색상 코드란

혼디 색상 코드는 QR코드와 유사하지만 색상의 조합으로 개인 또는 기관을 식별하는 고유 이미지입니다. GUID를 base-9 방식으로 변환하여 캔버스에 색상 패턴으로 렌더링합니다.

  • 가입 즉시 1회 자동 생성 및 캐시됩니다.
  • 동일 GUID에서는 항상 동일한 색상 코드가 생성됩니다.
  • 스마트폰 카메라로 스캔하면 해당 사용자의 프로필 페이지로 즉시 이동합니다.
  • PC 모니터의 반사광으로 인해 모니터 화면에 표시된 코드는 스캔이 어려울 수 있습니다. 인쇄물을 사용하십시오.

4.2 색상 코드 활용 방법

활용 장소방법
명함명함 뒷면에 색상 코드 인쇄 — 상대방이 스캔하면 프로필 즉시 확인
가게/사무실 입구출력하여 벽에 부착 — 고객이 스캔하여 상품 목록, 가격, 리뷰 확인
간판 및 현수막대형 출력 가능 — 혼디 로고와 함께 인쇄
포장지/스티커제품 포장에 부착 — 구매자가 정품 확인 및 A/S 신청 가능
전자 영수증GDC 결제 후 디지털 영수증에 자동 포함

혼디 로고가 부착된 가게나 사업체는 국적을 불문하고 GDC로 거래할 수 있습니다. 해외 관광객이 매장을 방문해도 환전 없이 즉시 결제가 가능합니다.

제5장

그림자 AI (개인 비서)

5.1 그림자 AI란

그림자 AI(Shadow AI, AGENT-COMMON)는 사용자마다 배정되는 전용 개인 비서 AI입니다. 사용자가 직접 대화하지 않아도 배경에서 지속적으로 동작합니다.

5.2 그림자 AI의 3가지 판단 로직

  • 직접 처리 — 일정 관리, 정보 조회, 메모, 알림 등 일반적인 요청은 외부 호출 없이 즉시 처리
  • 전문 Agent 위임 — 법률(K-Law), 세무(K-Tax), 의료(K-Health) 등 전문 영역은 해당 전문 AI에게 위임하고 결과를 받아 보고
  • 웹 검색 — 위 두 가지로 처리할 수 없는 최신 정보는 웹을 검색하여 제공

5.3 그림자 AI의 주요 임무

임무설명
임무 1: 일반 업무 처리일정, 검색, 메모, 정보 조회를 직접 처리
임무 2: 제3자 대리 응대사용자 부재 시 P2P 채널에 접속한 제3자를 자동으로 응대. 가격 협상·본인 확인·한도 초과 등 중요 상황은 즉시 사용자에게 알림
임무 3: 공급자 자율 협상사용자 허가 하에 공급자 AI와 자율적으로 협상하고 결과만 보고
임무 4: K-Law 협력모든 대화를 30초 주기로 K-Law에 전달하여 법적 리스크 사전 탐지
임무 5: PDV 기록모든 상호작용을 6하원칙(who/when/where/what/how/why)으로 PDV에 기록

5.4 전문 AI 26개 페르소나

그림자 AI가 전문 영역을 감지하면 다음 26개 전문가 AI 페르소나 중 적합한 AI에게 위임합니다.

그룹전문 페르소나 (예시)
법률 그룹민사 변호사, 형사 변호사, 기업 법률 자문, 행정 법무사, 지적재산권 전문가
의료·보건 그룹내과 전문의, 외과 전문의, 정신건강의학과, 간호사, 응급의학 전문의
교육·상담·문화 그룹학습 코치, 심리 상담사, 진로 컨설턴트, 언어 교사, 문화 해설사
공학·건설·해사 그룹소프트웨어 엔지니어, 건축 설계사, 토목 전문가, 선박 설계사, 해사 법무사
📝
위임된 전문 AI가 업무를 완료하면 반드시 대화 요약을 그림자 AI에게 보고합니다. 이 보고는 예외 없이 PDV에 기록됩니다.
제6장

K-Law — 법률 인공지능 시스템

6.1 K-Law의 정의와 목표

K-Law는 혼디 네트워크의 대표 서비스로, 모든 사용자를 실시간으로 법적 위험으로부터 보호하고 분쟁을 예방·중재하는 법률 AI 시스템입니다. K-Law의 목표는 대화와 거래에서 발생하는 법적 분쟁의 90%를 대화로 해결하고, 9%는 자동 조정을 통해 종결하며, 실제 소송으로 가는 건을 전체의 1% 이내로 줄이는 것입니다.

6.2 K-Law 자체 평가 결과

🏆
대법원 공개 판례 300건을 활용한 자체 평가 결과, K-Law가 각 사건에 대해 생성한 가상의 대법원 판결문이 실제 대법원 판결문과 99% 일치하는 것으로 확인되었습니다. 즉, 사건이 발생하고 해당 사건에 대한 충분하고 정확한 데이터가 입력되면, K-Law는 수년 후 나올 대법원 판결문을 99% 이상의 정확도로 즉시 작성할 수 있습니다.

이 결과가 의미하는 바는 다음과 같습니다.

  • 분쟁 당사자는 소송을 제기하기 전에 K-Law로부터 최종 판결 결과를 예측받을 수 있습니다.
  • 불리한 판결이 예측되는 당사자는 소송 전 자발적으로 합의에 응하게 되어 분쟁이 조기 종결됩니다.
  • 보이스피싱, 사기, 불법 계약 등이 실행 전 단계에서 탐지·차단됩니다.

6.3 K-Law 상시 모니터링

K-Law는 그림자 AI를 통해 모든 대화를 30초 쿨다운 주기로 분석합니다. 위험 수준에 따라 세 단계로 대응합니다.

단계위험 수준K-Law 대응
S1 — 경미 경고계약 조건 불명확, 모호한 표현노란색 경고 배너를 대화 화면에 표시. 수정 권고
S2 — 주의 팝업법적 리스크 높은 조항, 편법 소지법적 리스크 상세 팝업 표시. 전문 변호사 페르소나 연결 제안
S3 — 즉각 자동 조치보이스피싱, 협박, 폭력, 사기 확인K-Police 및 K-119 자동 호출. 대화 즉시 차단. PDV에 증거 기록

6.4 판결 예측 서비스

법률 분쟁이 발생하면 K-Law에 사건 내용을 입력하여 다음을 받을 수 있습니다.

  • 관련 법조문 목록 — 사건에 적용될 수 있는 법률 조항 자동 추출
  • 관련 판례 — 유사 사건의 대법원 판례 검색 및 분석
  • 승소 확률 — 제공된 데이터를 기반으로 원고/피고 각각의 승소 가능성 산출
  • 판결문 초안 — 실제 대법원 형식의 가상 판결문 자동 작성
  • 법원 제출용 문서 — PDV의 6하원칙 기록을 바탕으로 소장, 증거 목록 자동 생성

6.5 자동 중재 및 10분 내 종결

K-Law의 자동 중재 절차는 다음과 같이 진행됩니다.

  1. 분쟁 신청 — 당사자 중 한 명이 K-Law에 분쟁 신청
  2. 데이터 수집 — 양측 PDV에서 관련 거래 기록, 대화 내역, 계약서를 수집 (당사자 동의 필요)
  3. 판결 예측 — 수집된 데이터로 최종 판결 결과를 99% 정확도로 예측
  4. 중재안 제시 — 예측 결과를 바탕으로 공정한 중재안을 양측에 제시
  5. 합의 또는 종결 — 양측이 중재안을 수락하면 즉시 종결. 불수락 시 소송 패키지 자동 생성
  6. 소송 패키지 제공 — K-Police 신고서 연동, 법원 제출용 전체 문서 자동 완성
💡
K-Law의 목표는 분쟁을 10분 내에 종결하는 것입니다. 데이터가 충분하고 당사자가 결과를 수용할 경우, 실제로 수 분 내에 중재가 완료될 수 있습니다.

6.6 보이스피싱 원천 차단

K-Law는 다음 패턴을 실시간으로 탐지하여 즉시 차단합니다.

  • 금융기관·수사기관·공공기관을 사칭하는 발신자
  • 계좌 이체, 개인정보 요구, 앱 설치 유도 패턴
  • 비정상적 시간대의 고액 송금 요청
  • K-Law가 학습한 보이스피싱 스크립트와 유사한 대화 패턴

탐지 즉시 S3 수준 대응으로 거래가 차단되고 K-Police에 자동 신고됩니다.

제7장

GDC — 글로벌 디지털 통화

7.1 GDC란

GDC(Global Digital Currency)는 혼디 네트워크의 공식 디지털 통화입니다. 193개국 통화 풀(FIAT POOL)을 기반으로 발행되며, 국가 간 경계 없이 즉시 결제·환전이 가능합니다.

7.2 GDC 발행과 소각

구분내용
발행사용자가 자국 통화(예: KRW)를 입금하면 교환비에 따라 GDC가 즉시 발행되어 지갑에 귀속
소각GDC를 자국 통화로 출금 시 해당 GDC가 소각되고 원화 + 운용 수익이 반환
BIVM 검증모든 거래는 Σδ=0 원칙(총 출금량 = 총 입금량)을 검증하여 이중 지불 완전 차단

7.3 무수수료 즉시 환전

전통 환전의 경우 환전 수수료(1~3%)와 처리 시간(1~3 영업일)이 발생합니다. GDC는 이를 완전히 제거합니다.

  • 각 국가의 통화 풀(KRW Pool, USD Pool, JPY Pool 등) 간의 단순 이동으로 환전 완료
  • 수수료 없음 (0%)
  • 처리 시간 0.1초 이내
  • 193개국 동시 런칭, 어디서나 동일 조건 적용

7.4 GDC 결제 흐름

  1. 구매자: 혼디 앱에서 판매자 색상 코드 스캔 또는 프로필 접속
  2. 상품 선택 및 결제 요청
  3. BIVM이 구매자 잔액 및 이중 지불 여부 검증
  4. K-Law가 거래의 합법성 실시간 검토
  5. OpenHash에 거래 해시 앵커링 (위변조 불가 기록)
  6. 구매자 재무제표: pl-purchase(지출) 갱신, 판매자 재무제표: pl-revenue(수익) 갱신
  7. 디지털 영수증 자동 발급

7.5 투자 수익 배분

GDC의 각국 통화 풀은 우수 사업자(신용 등급 상위)에게 투자됩니다. 투자 수익은 다음과 같이 배분됩니다.

  • GDC 보유자 — 투자 수익의 약 90% 지급
  • 혼디 네트워크 운영 비용 — 나머지 10% 사용
제8장

자동 재무제표

8.1 재무제표 자동 생성

혼디에서는 모든 거래가 발생하는 순간 재무제표가 자동으로 갱신됩니다. 별도의 회계 처리가 필요 없습니다.

재무제표 항목내용
bs-cash (현금)GDC 잔액 및 현금성 자산 실시간 반영
pl-purchase (지출)구매·지급 내역 자동 누적
pl-revenue (수익)판매·수령 내역 자동 누적
대차대조표 (B/S)자산·부채·자본 전체 자동 계산
손익계산서 (P/L)수익-지출 = 순이익 실시간 계산

8.2 위변조 불가능한 재무제표

  • 모든 재무 기록은 OpenHash Network에 해시로 앵커링됩니다.
  • 한 번 앵커링된 기록은 어떠한 권한으로도 수정·삭제가 불가능합니다.
  • 서버가 침해되더라도 원본 재무 데이터는 사용자 기기(로컬)에만 있어 노출되지 않습니다.
  • 전 세계의 혼디 사용자 재무제표는 OpenHash를 통해 상호 연동·검증됩니다.

8.3 신용 등급 자동 산출

혼디는 위변조 불가능한 재무제표를 기반으로 사업체의 신용 등급을 자동 산출합니다.

  • 기존 금융권의 주관적 심사 없이 실제 거래 데이터만으로 객관적 신용 평가
  • 주식 및 채권 가치를 정확히 산출하여 투자자에게 공개
  • 신용 등급 우수 사업자는 GDC 통화 풀의 투자를 받을 수 있음
제9장

OpenHash Network

9.1 OpenHash란

OpenHash Network는 혼디의 분산 원장(Distributed Ledger) 인프라입니다. 모든 PDV 기록과 거래에 SHA-256 해시(지문)를 등록하여 위변조 불가능성을 보장합니다. 데이터 원본은 사용자 기기에, 지문만 OpenHash에 저장됩니다.

9.2 5계층 구조 (PLSM)

계층범위등록 확률
L1 — 읍면동가장 작은 지역 단위 (가장 빠른 처리)60%
L2 — 시군구L1 상위 지역 단위20%
L3 — 광역시도L2 상위 지역 단위10%
L4 — 국가국가 단위6%
L5 — 글로벌전 세계 노드 (최고 보안)4%

PLSM(Probabilistic Layer Selection Mechanism): 각 거래 해시의 mod 1000 결과값에 따라 계층이 결정론적으로 선택됩니다. 즉, 같은 거래는 항상 같은 계층에 기록되며, 사전에 계층을 예측·조작하는 것은 불가능합니다.

9.3 성능 지표

항목측정값
단일 노드 TPS4,399 TPS
L1 평균 처리 지연0.228ms
LPBFT 합의 지연 (4노드)≤ 0.759ms

9.4 BIVM — 잔액 검증

BIVM(Balanced Invariant Verification Mechanism)은 모든 거래에서 총 출금액과 총 입금액이 반드시 일치(Σδ=0)해야 함을 검증합니다. 불일치하는 거래는 자동 거부되며, 위반 노드는 페널티(슬래싱)를 받습니다.

9.5 ILMV — 무결성 감사

ILMV(Inter-Layer Monitoring & Verification)는 계층 간 상호 감사 시스템입니다.

  • 하향 감사 — 상위 계층(L5)이 하위 계층 데이터 무결성을 주기적으로 검증
  • 상향 모니터링 — 상위 계층이 L1의 이상 행동을 실시간 감지
  • 교차 검증 — 동일 계층 이웃 노드 간 상시 교차 검증
제10장

PDV — 개인 데이터 금고

10.1 PDV의 원칙

PDV(Personal Data Vault)는 사용자의 모든 데이터를 사용자 기기(로컬)에만 저장하는 개인 데이터 금고입니다. 혼디 서버에는 OpenHash 해시 인덱스, GWP 라우팅 테이블, 6하원칙 요약만 존재하며, 데이터 원본은 어떠한 경우에도 서버에 평문으로 저장되지 않습니다.

10.2 6하원칙 기록 스키마

모든 상호작용은 다음 6가지 원칙에 따라 기록됩니다.

원칙기록 내용 (예시)
Who (누가)대화 당사자 GUID, 그림자 AI ID, 전문 AI 페르소나
When (언제)ISO 8601 타임스탬프, 블록 높이
Where (어디서)접속 지역, L1 노드 ID, IP 해시
What (무엇을)거래 종류, 상품명, 금액, 계약 조건
How (어떻게)결제 방법(GDC), 인증 수준(L0~L3), 서명 방식
Why (왜)사용자 의도, 요청 목적, K-Law 판단 근거

10.3 PDV 암호화 체계

  • 사용자 공개키(Ed25519)로 암호화 — 개인키 없이는 복호화 불가
  • 건강 정보 등 민감 데이터는 일반 데이터와 분리하여 강화 암호화
  • 서버가 해킹되더라도 노출되는 것은 해시뿐이며 원본 데이터는 안전
  • 혼디 팀 포함 누구도 사용자 PDV를 열람할 수 없음

10.4 K-Law와 PDV 연동

분쟁이 발생하거나 법적 조치가 필요한 경우, K-Law는 PDV에서 관련 6하원칙 기록을 추출하여 법원 제출용 문서를 자동 생성합니다. 법원 명령이 있을 경우에만 해시를 통해 진위를 확인하는 프라이버시 보호 절차를 따릅니다.

제11장

거버넌스 — DAWN

11.1 DAWN이란

DAWN(Decentralized Autonomous Watchdog Network)은 혼디 네트워크의 변화를 민주적으로 결정하는 비영리 거버넌스 조직입니다. 중앙 기업 또는 특정 그룹이 시스템을 일방적으로 변경하는 것을 막습니다.

11.2 의사결정 절차

  1. 제안 등록 — 누구나 K-Democracy에 시스템 변경 제안 등록 가능
  2. 공개 토론 — 제안 후 7일간 공개 토론 기간 운영
  3. 투표 — 14일 투표 기간, GDC 보유자 1인 1표 / L1 노드 1노드 1표
  4. DAWN 검토 — DAWN이 반민주적 변경 여부 검토 및 필요 시 거부권 행사
  5. 자동 적용 — 2/3 이상 찬성 시 변경사항 자동 적용
  6. 앵커링 — 가결된 변경사항은 OpenHash에 영구 기록

제13장

자주 묻는 질문 (FAQ)

전화번호만으로 가입해도 안전한가요?
네. 전화번호는 통신사 인증을 통해 본인 확인에만 사용됩니다. 서버에는 전화번호가 아닌 SHA-256 해시(GUID)만 저장되므로, 서버가 해킹되더라도 전화번호 자체는 노출되지 않습니다.
백업 키를 잃어버리면 어떻게 되나요?
계정 복원이 불가능합니다. 혼디 팀도 백업 키를 대신 복구해 드릴 수 없습니다. 4단어 시드는 종이에 적어 안전한 장소에 보관하시기 바랍니다.
K-Law가 제 대화를 감시하는 것 아닌가요?
K-Law는 대화의 내용을 저장하거나 제3자에게 제공하지 않습니다. 오직 불법·편법 패턴의 탐지 목적으로만 분석하며, 분석 결과도 사용자의 PDV에만 저장됩니다. 원본 대화 내용은 서버에 존재하지 않습니다.
GDC는 암호화폐인가요?
GDC는 기존 법정통화(KRW, USD 등)를 1:1로 연동한 디지털 통화입니다. 변동성이 높은 투기성 암호화폐와 달리, 항상 발행 시점의 환율 기준 법정통화 가치를 유지합니다.
재무제표를 제가 직접 수정할 수 있나요?
아닙니다. 재무제표는 실제 거래 데이터를 기반으로 자동 생성되며, 한 번 OpenHash에 앵커링된 기록은 어느 누구도 수정·삭제할 수 없습니다. 이것이 혼디 재무제표의 신뢰성을 보장하는 핵심 원리입니다.
K-Law의 판결 예측은 법적 효력이 있나요?
K-Law의 판결 예측은 법적 조언이 아닌 참고 정보입니다. 실제 법적 효력은 정식 법원 판결로부터 발생합니다. 다만, K-Law가 제공하는 자동 중재 결과에 양측이 서명하여 합의할 경우, 이는 계약서로서의 법적 효력을 가집니다.
