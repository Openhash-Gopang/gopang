## 0. 요약

혼디는 카카오톡·텔레그램과 같은 채팅 앱 형태를 취하지만, 대화 상대의 주축이 사람이 아니라 **AI 비서(AC, AGENT-COMMON)**입니다. AC는 사용자의 요구(배달 주문부터 이혼소송, 개인파산 신청, 등본 발급, 위치기반서비스 등록까지)를 분석해 수천 종의 전문 AI(SP, System Prompt 기반 에이전트)와 수백 개 기관 SP를 적절한 순서로 호출·조율하는 **평생 집사이자 법률 전문가** 역할을 수행합니다.

본 문서는 이 아키텍처를 운영하는 데 드는 실제 비용(LLM 추론 비용, 인프라 비용)과, 제주 필드테스트 및 전국 사업 확장 단계의 손익 모델·과금 체계를 정리합니다.

## 1. 아키텍처 요약 — AC가 통괄하는 이유

기존에는 "절차와 조직 중 무엇이 먼저인가"라는 질문에서 출발해, 다음 원칙에 도달했습니다.

- **절차(procedure)가 조직(organization)보다 우선한다.** 목표를 달성하기 위한 절차 시퀀스를 먼저 정하고, 그 절차의 각 단계를 채우는 재료로서 기관이 동적으로 붙는다.
- 오케스트레이터는 **도메인에 무관**해야 한다. "법률 요청인가"가 아니라 "다기관·다절차 조합이 필요한 요청인가"가 게이트 기준이다.
- 따라서 통괄 권한은 도메인 전문 SP(Lawyer 등)가 아니라 **AC**에 있다. Lawyer, 세무사, 의사 등 EXPERT SP는 판단이 필요한 개별 단계에서만 호출되는 하위 실행자로 역할이 축소된다.

### 1-1. 레지스트리 구조

| 레지스트리 | 역할 |
|---|---|
| `PROCEDURE_REGISTRY` | 목표 → 절차 시퀀스 캐시 (예: 개인파산 신청 = 6단계 시퀀스) |
| `ORG_REGISTRY` | 개별 기관(수백 개)의 상호작용 방법 — 사법부/행정부/공공기관/민간지정기관으로 유형 분류 |
| `EXPERT_REGISTRY` | 판단 노드에서 호출되는 전문가 SP (변호사, 세무사, 의사 등) |
| `CASE_TRACKER` | 다세션(수개월 소요) 절차의 진행 상태 저장 |

### 1-2. 인간 경계 원칙

모든 절차의 최종 제출·서명·인증 단계는 기본적으로 인간 전속(`human_gate=true`)입니다. AI는 서류 조합까지만 수행합니다. 이는 법률 서비스 규제 리스크(변호사법 34조, 로톡 8년 분쟁 사례)를 구조적으로 회피하기 위한 설계이기도 합니다.

## 2. 원가 구조 — LLM 추론 비용 실측 기준

### 2-1. 기준 단가 (2026-07 기준, DeepSeek API 공식 가격)

| 항목 | 단가 (USD) |
|---|---|
| DeepSeek V4 Flash — input (cache miss) | $0.14 / 1M tokens |
| DeepSeek V4 Flash — input (cache hit) | $0.0028 / 1M tokens (98% 할인) |
| DeepSeek V4 Flash — output | $0.28 / 1M tokens |
| DeepSeek V4 Pro — input (cache miss) | $0.435 / 1M tokens |
| DeepSeek V4 Pro — output | $0.87 / 1M tokens |

환율 가정: ₩1,400 / $1 (집행 시점 실제 환율로 교체 필요)

두 모델 모두 MIT 라이선스로 자체 호스팅이 가능합니다 (HuggingFace: `deepseek-ai/DeepSeek-V4-Flash`, `deepseek-ai/DeepSeek-V4-Pro`).

### 2-2. 요청 유형별 실측 원가 추정

| 요청 유형 | 모델 | 토큰 가정 | 건당 원가(₩) |
|---|---|---|---|
| A. 일상대화 1턴 (Tier 0) | Flash | in 500(80%캐시)+out 150 | ≈ 0.08 |
| B. 단순 SP 위임 1건 (배달주문·등본안내) | Flash | in 2,000(50%캐시)+out 400 | ≈ 0.36 |
| C. EXPERT 자문 1건 (법률·세무 판단) | Pro | in 8,000(30%캐시)+out 2,000 | ≈ 6 |
| D. 오케스트레이션 1건 (개인파산 전체, ~20회 체인) | 혼합 | C×20×여유2배 | ≈ 240 |
| E. K-Law 미래판결예측 | Pro | in 50,000(20%캐시)+out 5,000 | ≈ 31 |

**핵심 발견**: 가장 비싼 단일 요청(K-Law 판결예측)도 원가 ₩31 수준입니다. DeepSeek이 이미 초대규모 배치로 원가를 극한까지 낮춰뒀기 때문에, **토큰 비용은 가격 설계의 병목이 아닙니다.** 손익분기의 실제 변수는 고정비(인프라·운영)입니다.

### 2-3. 자체 호스팅 vs API — 국내 전국 사업 확장 시 비교

| 항목 | 값 |
|---|---|
| H200(141GB) 클라우드 임대 시세(중앙값) | $3.5~4.0/GPU·시간 |
| H200 1대 월 임대비(₩) | 약 3,600,000~4,000,000 |
| 단일 H200 월 처리 가능 토큰(추정, 미검증) | 약 25억~80억 토큰 |
| 자체 호스팅 토큰당 원가(역산) | 약 ₩1,440/M토큰 |
| API 블렌디드 평균 원가 | 약 ₩250~400/M토큰 |

**결론**: 단일~소수 GPU 규모에서는 자체 호스팅이 API보다 3~5배 더 비쌉니다. API가 손익분기를 맞추려면 GPU 1대가 월 약 144억 토큰을 처리해야 하는데, 이는 단일 GPU 현실적 처리량을 크게 초과합니다. **"비용 절감"을 이유로 자체 서버 전환을 서두를 근거는 약합니다.** 자체 호스팅을 고려할 이유는 원가가 아니라 **데이터 주권**(OpenHash 철학)과 민감정보(법률·의료) 처리 규제 대응입니다 — 이 경우 하이브리드(민감 트래픽만 분리)가 합리적입니다.

## 4. 2단계 — 전국 사업 확장

### 4-1. Tier별 가격 제안

| Tier | 내용 | 원가(API 기준) | 권장가 | 근거 |
|---|---|---|---|---|
| Tier 0 | 일상대화·캐시된 절차안내 | ₩36/월/인 | **무료** | 사용자 획득비용으로 흡수 |
| Tier 1 | EXPERT 자문 1건 | ₩6/건 | **₩3,000~5,000/건** | 인간 변호사 초회상담료(5~10만원)의 1/10~1/20 |
| Tier 1 | K-Law 판결예측 1건 | ₩31/건 | **₩15,000~30,000/건** | 프리미엄 상품, 소송비용 대비 저렴함으로 포지셔닝 |
| Tier 2 | 월 구독(크레딧 포함) | — | **₩9,900/월** | 카카오톡 대비 진입장벽 최소화 |
| Tier 3 | B2B (K-Market/전문직 광고형) | — | 거래 3% + 정액 노출료 | 성사수수료 없음 — 변호사법 34조 리스크 회피 (로톡 사례 참조) |

### 4-2. 법률 서비스 과금 원칙 (규제 리스크 대응)

- AI 판단(Lawyer SP 자문, K-Law 시뮬레이션)과 **인간 변호사 연결**은 반드시 분리 과금합니다.
- 인간 변호사 연결은 성사수수료 없이 **정액 광고 노출료**만 허용합니다 — "중개형"이 아닌 "광고형"으로 유지해야 변호사법 34조 위반 소지를 피할 수 있습니다.
- 무료지원(법률구조공단 등) 대상자는 유료 EXPERT 안내보다 우선 고지합니다.

## 5. 미확정·검증 필요 항목 (정직하게 남겨둠)

1. **Oracle Cloud 실제 청구서**: §3-1의 고정비는 추정치입니다. 실측값으로 교체 필요.
2. **단일 GPU 실측 처리량**: §2-3의 "25억~80억 토큰/월"은 공개 벤치마크 부재로 인한 넓은 추정 범위입니다. vLLM/SGLang 기반 실제 부하테스트 필요.
3. **Tier 1 지불의향(WTP) 검증**: ₩3,000~5,000/건 가격이 실제 시장에서 받아들여지는지 제주 파일럿 A/B 테스트로 검증 필요.
4. **환율 변동**: ₩1,400/$1 가정은 집행 시점 실제 환율로 재계산 필요.
5. **광고형 노출료의 정확한 허용 범위**: 변호사법 34조 해석은 사업적·법률적 검토가 추가로 필요한 영역입니다.
