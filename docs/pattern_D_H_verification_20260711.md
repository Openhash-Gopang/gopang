## 먼저 밝혀진 구조 — "K서비스 22개"와 "메인 포털"은 다른 층입니다

이번 재검증 중 가장 중요하게 확인된 사실: `src/gopang/ai/call-ai.js`(EXPERT 태그,
K-Search 실행부, GWP 태그 디스패처 등 오케스트레이션 로직 전체)를 22개 K서비스
저장소 중 단 하나도 참조하지 않습니다. 이건 결함이 아니라 **의도된 2계층 구조**입니다:

- **gopang 모노레포의 루트 `webapp.html`** = hondi.net 메인 포털(개인 AI 비서,
  AGENT-COMMON 그림자 AI). 1000개 시나리오(패턴 A~I)가 다루는 게 전부 이 층입니다.
- **22개 K서비스 저장소**(tax.hondi.net, health.hondi.net 등) = 각자 독립된
  단일목적 챗봇. 메인 포털이 이들을 "사용자 요구를 실현하는 수단"으로 호출·통괄합니다
  (주피터님이 확인해 주신 원칙 그대로).

메인 포털이 K서비스를 호출하는 방법은 두 가지로 갈립니다:
1. **같은 창에서 시스템 프롬프트 교체**(패턴 D류) — 새 탭 없음.
2. **`_gwpLaunch()`로 새 탭을 열며 URL 파라미터(`ctx`)로 대화 맥락을 실어 보냄**
   (패턴 H류 포함, `gwp_registry`의 `type:'tab'` 항목 5개).

## 패턴 D(EXPERT) — ✅ 정상 확인(지난 세션 결론 유지)

`src/gopang/ai/expert-session.js`의 `handleExpertTag()`는 실제로 존재하고 완성도 있게
구현돼 있습니다 — 같은 창 안에서 `CFG.system`을 전문가 페르소나 프롬프트로 교체하는
방식이라(탭을 새로 열지 않음) 패턴 H류가 겪는 "핸드오프 맥락 유실" 문제가 구조적으로
발생하지 않습니다. 종료 조건(명시적 종료 발화·10분 타임아웃)과 종료 시 PDV 6하원칙
요약 기록까지 구현돼 있음을 확인했습니다. **재검증 결과도 정상.**

## 패턴 H(kbusiness) — 🔴 실제 결함 발견(지난 세션 "정상" 결론 정정)

`gwp_registry`의 `type:'tab'` 항목 5개(kemergency→911, kgdc→gdc, kcommerce→market,
jeju→jeju, kbusiness→market의 `kmarket_admin_dashboard.html`) 각각의 **목적지 저장소를
전부 직접 클론해 `ctx` 파라미터 수신 여부를 대조**했습니다:

| GWP id | 목적지 파일 | ctx 수신 처리 | 상태 |
|---|---|---|---|
| kemergency | 911/webapp.html | 있음(`ctx_enc` 분기 확인) | ✅ 정상 |
| kcommerce | market/webapp.html | 있음(자동 입력창 채움+전송) | ✅ 정상 |
| jeju | jeju/webapp.html | 있음(`ctx_enc=b64` 분기 확인) | ✅ 정상 |
| kgdc | gdc/webapp.html | 없음 — 그러나 **의도된 설계**(코드 주석: "gdc는 대화형이 아니라 거래형 UI") | ✅ 정상(예외) |
| **kbusiness** | market/**kmarket_admin_dashboard.html** | **전혀 없음** | 🔴 **결함** |

`kbusiness`만 다섯 중 유일하게 깨져 있었습니다 — 같은 `market` 저장소 안에서 메인
`webapp.html`은 `ctx`를 정상적으로 받는데, GWP 목적지로 등록된 별도 파일
`kmarket_admin_dashboard.html`(원래 판매자용 정적 관리자 대시보드로 먼저 만들어지고,
2026-07-05에 "AI 경영 어드바이저" 챗 패널이 나중에 얹힌 파일)에는 그 로직이 아예
없었습니다. `_buildBizContext()`는 오직 `ADMIN_DATA`(대시보드 자체 데이터)만 근거로
컨텍스트를 조립하고, 메인 포털이 `_gwpLaunch()`로 실어 보낸 `ctx`(사용자가 원래 무엇을
물었는지)는 URL에 도착은 하지만 **아무도 안 읽습니다**.

### 실제 영향
사용자가 hondi.net 메인 포털에서 "우리 가게 부가세 신고 언제까지야?"라고 물어 GWP가
K-Business 탭을 열어도, 그 질문은 새 탭에 전달되지 않고 사라집니다. 사용자는 새로
열린 대시보드에서 채팅 버튼을 직접 눌러 처음부터 다시 입력해야 합니다 — "AI가 알아서
넘겨준다"는 GWP의 설계 의도가 이 경로에서만 깨져 있었습니다.

### 수정
`market_fix.diff`로 제공합니다 — 같은 저장소의 `webapp.html`이 이미 쓰고 있는 것과
동일한 `ctx`/`ctx_enc` 수신·복원 패턴을 `kmarket_admin_dashboard.html`에도 추가해,
페이지 로드 시 자동으로 채팅 패널을 열고 받은 질문을 자동 전송하도록 했습니다.
