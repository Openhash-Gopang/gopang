# 혼디(Hondi, 구 Gopang) 시스템 통합 테스트 체크리스트

> 작성: 2026-06-27(갱신) · 대상: 혼디 백서(`docs/gopang_whitepaper.md`) + 16개 K-서비스 저장소 직접 검증 + 전문 분야(기관) AI 15개·전문가 AI 26개 페르소나 개별 검증
> 총 항목 수: **171개** (섹션 14개)

## 갱신 내역(2026-06-27)

- 전문 분야(기관) AI 15개 도메인, 전문가 AI 26개 페르소나를 각각 **개별 행**으로 분리(신규 E-1·E-2 섹션)
- GitHub PAT으로 16개 K-서비스 저장소를 직접 클론하여 GWP_DONE/GWP_MESSAGE 구현 여부를 실측 — **16개 중 market만 구현 확인**(나머지는 폴백 경로로 검증해야 함)
- K-Law 백그라운드 모니터(30초 쿨다운)는 구현 확인, **CRITICAL 시 K-Police/K-119 자동 호출은 미구현 확인**(F04)
- 기존 E섹션의 K-서비스별 개별 항목(E06~E19)은 E-1로 대체 — **ID 공백은 의도된 것**(과거 체크 상태와의 혼선 방지)

## 사용 방법

- 테스트 수행자는 `통과여부` 칸에 `☑ 통과` 또는 `☑ 실패`로 표시하고, `비고`에 재현 절차·로그·스크린샷 경로를 남깁니다.
- 실패 항목은 별도 이슈로 등록 후 이 문서의 `비고`에 이슈 링크를 남깁니다.
- 백서가 "설계 사양"으로만 기술하고 현재 코드베이스에서 구현이 확인되지 않은 항목은 `기대 결과` 칸에 ⚠️/❌로 미리 표시되어 있습니다(검증 시점: 2026-06-27).
