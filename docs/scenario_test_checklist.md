## 3차 갱신 내역(2026-06-27)

1. **테스트 실행 순서를 알파벳 순서 → 의존관계 순서로 재배열**했습니다(아래 "권장 테스트 순서" 참고). 문서 안 섹션 ID(A/B/C...)는 이전 버전과 호환되도록 그대로 유지했고, **표시 순서만** 바꿨습니다.
2. 각 항목에 **테스트 방법 / 관련 코드 / 결과 / 추후 갱신 계획** 4칸을 추가했습니다. `관련 코드`는 이번 세션에서 직접 확인한 범위 내에서 선기입했고, 모르는 곳은 빈칸으로 남겨 실제 테스트하며 채우도록 했습니다.
3. **E-2(전문가 AI 26개 페르소나)를 문서 맨 뒤로 이동**했습니다 — AGENT-COMMON 프롬프트에 아직 `[EXPERT:id]` 출력 지침이 없어 정상 흐름으로는 트리거되지 않기 때문입니다(차단됨/보류).
4. **K(관리자 정리)와 L(UI 회귀)의 순서를 바꿔 L을 먼저** 배치했습니다 — 테스트 계정이 살아있는 상태에서 UI를 확인한 뒤, 마지막에 K로 정리(일괄삭제)하는 것이 합리적입니다.
