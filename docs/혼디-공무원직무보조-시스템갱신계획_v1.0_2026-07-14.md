# 혼디 공무원 직무 보조 기능 추가 — 시스템 갱신 계획 (v1.0)

작성일: 2026-07-14
원본: `공무원_업무지시_100건_해법_PDV완전보급_가정.md`
상태: **확정판** — 100건 전 항목 실제 상태(해결됨/조건부/별도경로/구조적불가) 확인 후 트랙별 배치 완료

## 1. 목표

Gopang의 기존 SP 계층(jeju-router.js → 13개 국 → 122개 과 → 184개 읍면동/팀, 515개 SP, GOV-TIER-IO-SCHEMA 준수)에 "정부24 조회 + PDV 동의" 공통 능력을 추가하여, 공무원이 시민 개인정보를 요하는 업무를 담당 SP 내에서 즉시 처리할 수 있게 한다. 신규 시스템 구축이 아니라 **기존 SP 계층의 능력 확장**이 원칙이다.

## 4. 트랙 3 — 기관 내부 SaaS (10건, 별도 트랙)

39·40·41·42·43·44·45(공용차량 배차, 회의실 예약, 공문서 기안·결재, 예산 집행률 경고, 초과근무 관리, 비품관리, 감사지적 이행) + 89·91·92(공용예산 카드 점검, 정보공개청구 처리기한, **조회 로그 감사 — 92번은 트랙1/2 전체의 안전장치로 특히 중요**)

이 10건은 PDV 동의 절차가 필요 없다(시민 개인정보가 아님). 이전 세션 ③항목 결론 그대로: 새올·e호조·온나라시스템에 기관용 AC를 직접 연동하는 문제이며, **DeepSeek 국내 파인튜닝 + 망분리 등급별 배포**가 선행 과제다. 이번 갱신 계획(트랙1/2, PDV 동의 SDK)과는 독립적으로 진행 가능.

**92번(조회 로그 감사)은 예외적으로 트랙1/2와 강하게 연결된다** — 트랙1/2의 레이어 A(PDV 동의 SDK)가 만들어내는 모든 동의·조회 이벤트가 바로 이 감사 로그의 원천 데이터이기 때문이다. 구현 순서상 레이어 A 설계 시점에 92번의 로그 스키마를 함께 정의해야 한다.
