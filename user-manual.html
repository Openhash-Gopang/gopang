사용자 매뉴얼
혼디 시스템 안내
본 웹앱(베타)은 정식 안드로이드·아이폰 앱 출시에 앞서 핵심 기능을 미리 검증하기 위한 기능 실증(Proof of Concept) 목적의 버전입니다. 정식 앱 출시 전까지 일부 기능이 변경되거나 일시 중단될 수 있습니다.
기본 사용법
좌우 가장자리 손잡이
화면 좌우 가장자리를 당기면(스와이프) 좌우 메뉴가 표시됩니다. 가장자리에 작은 손잡이가 표시되어 있으니, 그 부분을 당겨도 되고 직접 탭해도 열립니다.
AI 비서 설정
상단의 AI 버튼은 원하는 시점에 한 번만 설정하면 됩니다. 버튼을 누르면 OpenRouter API Key 등록 화면으로 이동하고, Key를 한 번 등록해두면 그 이후로는 AI 버튼이 계속 활성 상태로 유지됩니다. Key가 없으면 AI 버튼은 항상 비활성 상태입니다.
설정 창(바퀴모양 아이콘) — Profile
설정 창(바퀴모양 아이콘)의 Profile은 내부용과 공개용 두 블럭으로 구성됩니다.
공개용 블럭의 데이터는 곧 사용자의 웹 사이트입니다. 가령, 판매자는 자신의 상품과 서비스를 등록하고, 구매자는 자신의 구매 이력을 기록합니다.
마이크 두 가지 — 1회용과 통화형
입력창 주변에는 마이크가 두 개 있으며, 동작 방식이 다릅니다.
① 왼쪽 가장자리 마이크 — 한 번 말하면 끝나는 1회용입니다.

② 입력 필드 안쪽 마이크(전송 버튼 왼쪽) — 누르면 전화 통화처럼 대화창을 떠나기 전까지 계속 듣습니다. 한 번만 누르고 편하게 이야기하면 됩니다.
②번 마이크가 켜진 상태에서 말을 잠깐 멈추고 직접 키보드로 입력칸을 수정해도 내용이 지워지지 않습니다. 멈췄던 지점 뒤에 이어서 받아쓰기가 계속되므로, 타이핑과 음성 입력을 자유롭게 섞어 쓸 수 있습니다. 마이크를 끄려면 같은 버튼을 다시 누르세요.
대화 상대를 정하지 않았다면
대화 상대를 지정하지 않고 대화를 시작하면, 자동으로 '나만의 AI 비서'가 호출됩니다.
그룹 대화방
그룹 대화방은 방장이 대화방을 개설하고, 상대방을 초청해야 합니다.
부재 중 대화
부재 중 대화는 좌측 패널의 메시지 수신함에 저장됩니다.
신원·보안
경비 AI Agent — 불법 행위 차단
보이스피싱 등 불법 행위는 '경비 AI Agent'에 의해 차단되고, 신고 등 적절한 조치가 자동으로 실행됩니다.
휴대폰 번호 — 전 세계 유일한 식별자
내 휴대폰 번호는 전 세계에서 유일하며, 고유합니다. 그러므로 나를 식별하는 최적의 데이터이며, 혼디 가입 시 전화번호를 입력하는 이유입니다.
혼디는 글로벌 시스템이며, 디폴트 국가명은 한국입니다.
로그인 키와 GUID
가입 시 이 기기 안에서만 암호키(개인키)가 만들어집니다. 이 키는 서버로 전송되지 않으며, 혼디는 이 키의 사본을 갖고 있지 않습니다. 전화번호로 만들어지는 GUID가 계정의 이름표라면, 이 개인키는 그 이름표의 진짜 주인임을 증명하는 도장입니다 — 로그인할 때마다 이 키로 서명해 본인임을 증명합니다.
그래서 전화번호나 닉네임을 다른 사람이 알아도 로그인할 수 없습니다. 오직 이 개인키를 가진 기기만 로그인할 수 있습니다.
백업 키 — 분실 시 유일한 복구 수단
위에서 말한 개인키는 이 기기에만 있고, 혼디 서버 어디에도 사본이 없습니다. 그러므로 기기를 분실하거나 초기화하면, 백업 키를 미리 저장해두지 않은 이상 같은 계정을 되찾을 방법이 없습니다. 전화번호를 다시 입력해도 소용없습니다 — 그건 이름표일 뿐, 도장이 아니기 때문입니다.
가입 직후 백업 키를 한 번 보여드리며 저장을 요청합니다. 이후에도 설정 → 백업 키에서 언제든 다시 꺼내보고, 새 기기에 입력해 계정을 옮길 수 있습니다.
이 키를 아는 사람은 누구나 그 계정이 될 수 있으므로, 캡처하거나 다른 사람과 공유하지 말고 본인만 아는 안전한 곳(비밀번호 관리자, 본인 이메일 등)에 보관하십시오.
IPv6 주소 — 웹과의 통합
모든 사용자는 고유한 IPv6 주소를 갖습니다. 이 주소는 향후 웹과 혼디의 통합을 위한 것입니다.
PDV & OpenHash
영구 보존 — 언행에 신중하십시오
대화 창에서 오간 모든 대화와 문서는 영구 보존되며, OpenHash 기술로 위변조를 원천 차단합니다. 그러므로 언행과 문서 등에 신중하십시오.
AI가 아니라 OpenHash
혼디의 가장 큰 특징은 AI가 아니라, Openhash입니다. 나만의 AI 비서는 나의 일상과 업무를 분 단위로 내 PDV에 기록하며, 이 기록은 위변조가 불가능합니다.
나만의 PDV (Private Data Vault)
나만의 AI 비서는 나의 일상과 업무, 대화와 주고받은 문서를 모두 '나만의 PDV'에 저장합니다. PDV는 나 자신만 접근할 수 있는 개인 데이터 금고입니다.
서버가 아니라 내 스마트폰에
모든 대화는 메시징 서버가 아니라, 내 스마트폰에 있는 '나만의 PDV'에 암호화되어 기록됩니다. 그러므로 데이터가 대량으로 유출되는 사고는 원천 차단될 뿐 아니라, 오직 나만 접근할 수 있습니다.
그러나 귀하의 언행과 문서는 귀하의 PDV뿐 아니라, 상대방의 PDV에도 함께 기록된다는 점을 잊지 마십시오.
혼디의 모든 데이터는 서버가 아니라 각자의 단말기에 보관되며, 서버는 사용자와 사용자를 연결하는 기능과 Openhash Network의 데이터 무결성 보장 장치로 역할합니다. 간단히, 혼디 서버들은 사용자 단말기에 저장된 수많은 데이터(대화, 문서 등)의 지문을 보관하는 장소입니다.
Openhash Network를 구성하는 서버들은 읍면동 단위로 하나씩 위치합니다.
OpenHash — 위변조 불가능한 지문
PDV는 OpenHash Network에 데이터 각각의 지문(Hash)을 보관합니다. 이 지문은 결코 위변조될 수 없습니다. Openhash의 주된 용도는 언제 어떤 일이 있었는지 입증하는 것입니다.
PDV 기록 삭제하기
내 기록을 삭제하는 방법은 통상의 파일과 같습니다. 내 PDV에 접속하여 삭제하면 됩니다.
단, 삭제된 데이터는 과거 사건의 진실성을 입증할 수 없습니다.
분쟁 해결 — 1초 안에 진실 확인
분쟁이나 갈등이 발생하면 PDV는 증거나 증언을 대체하며, 데이터의 진실성을 100% 보장합니다. 확인 과정은 1초 이내에 완료됩니다.
나만의 AI 비서
우측 패널 — 혼디의 하위 시스템
우측 패널은 나만의 AI 비서가 호출하는 전문 분야 AI Agent 목록입니다. 각 하위 시스템은 자체 웹 사이트에서 해당 기능을 상세히 설명합니다. 가령 K-Law는 법률 시스템으로, 나만의 AI 비서에게 법률 질의나 업무를 요청하면 나만의 AI 비서가 K-Law를 호출합니다. K-Law는 소송을 접수하기 전, 4~5년 뒤에나 보게 될 대법원 판결문을 신뢰할 만한 정확도로 미리 보여줍니다.
호출 즉시 대화창 우측 상단 제목이 "혼디"에서 "K-Law"로 바뀝니다. 즉 대화 상대방이 나만의 AI 비서에서 법률 전문 AI Agent로 넘어가는 것입니다. 대화가 끝나면 법률 Agent는 6하 원칙(누가·언제·어디서·무엇을·어떻게·왜)에 따라 나만의 AI 비서에게 대화 내용을 보고합니다.
System Prompt = 내비게이션
나만의 AI 비서를 포함해, 수십 종의 전문 분야 AI Agent는 모두 System Prompt라 불리는 지시에 의해 동작합니다.
Claude, DeepSeek, ChatGPT 등 다양한 LLM을 자동차에 비유하면, System Prompt는 내비게이션입니다. 차종은 달라도 내비게이션 지도가 같다면, 모든 차량의 주행 경로와 방법은 동일한 것과 같습니다.
왜 '나만의' AI 비서인가
나만의 AI 비서는 내 지시를 이행할 기능(Type A)과 타인이 나에게 요청한 업무를 내 대신 수행할 기능(Type B)을 모두 갖습니다. AI 비서가 '나만의' 비서인 이유는 시간이 지날수록 귀하의 일상과 업무에 특화되어, 1년쯤 뒤에는 '척하면 착하고 알아듣는' 귀하만의 비서로 진화하기 때문이며, 그 진화의 구체적 형식은 System Prompt입니다.
스스로 진화하는 비서
System Prompt는 귀하가 직접 수정·갱신할 수도 있지만, 나만의 AI 비서가 스스로 자신을 주기적으로 갱신합니다. 즉, 나만의 AI 비서는 스스로 진화하도록 설계되었습니다.
마치, 스스로 성장하고 수리하는 로봇과 같습니다.
커뮤니티·운영
개선 제안하기
혼디 시스템의 개선이나 수정을 위한 제안은 나만의 AI 비서에게 말씀해주세요. 그가 우리(개발자 그룹 팀 주피터)에게 전달합니다.
오픈소스 — Github 공개
혼디의 모든 소스코드는 Github에 공개되므로, 누구나 혼디의 시스템 개발과 갱신에 참여할 수 있습니다.
경제 시스템
운영 원칙과 혼디장
혼디는 수익자 부담 원칙에 따라 운영됩니다. 혼디 자체는 비영리 공동체이지만, 혼디의 장터인 '혼디장'은 영리를 위한 공간입니다. 누구나 혼디장에 상품을 등록하고 판매할 수 있습니다.
모두를 위한 자동 재무제표
혼디는 모든 개인과 단체(사업체 포함)의 재무제표를 자동으로 생성하고 갱신합니다. 5천만 한국인의 재무제표는 상호 연동되므로, 분식이나 회계 부정은 원천 차단됩니다.
GDC — 혼디의 결제 통화
GDC(Global Digital Currency)는 혼디의 결제 통화입니다. 혼디에서 상품과 서비스를 구매하려면 먼저 GDC로 환전해야 합니다.
다국적 통화 풀
혼디는 세계 193개국에서 동시에 런칭하므로, 다국적 통화 풀(Currency Pool)을 구성합니다. 풀의 자산인 각국 화폐는 각국 인덱스 주식과 채권 등에 투자되며, 그 수익은 대부분 GDC 보유자에게 환원되고 일부는 혼디 시스템 운영 비용으로 충당됩니다.
혼디 코드란?
혼디 코드는 QR코드를 대신하는 혼디만의 시각 식별 코드입니다. "혼디"라는 글자 자체에 색상을 입혀 개인·가게 고유의 식별 정보를 담습니다.
"혼디"의 ㅣ(이응) 획을 10칸(v1) 또는 20칸(v2)으로 나누어, 각 칸에 9가지 색상 중 하나를 배정합니다. 칸의 배색 조합이 곧 내 고유 번호입니다.

ㅎ는 파랑, ㅗ는 빨강, ㄴ·ㄷ은 검정으로 고정되어 있어 스캐너가 글자를 인식하는 기준점(calibration) 역할을 합니다.
혼디 코드 발급 — 가입하면 자동 생성
혼디 코드는 가입과 동시에 자동으로 만들어집니다. 별도 신청이나 비용이 없습니다.
① 가입 완료 → 내 GUID(고유 식별 번호)로부터 혼디 코드가 즉시 계산됩니다.

② 이후 영구 고정 → 닉네임·프로필·주소를 변경해도 코드는 바뀌지 않습니다. 한 번 인쇄해두면 평생 유효합니다.
혼디 코드 조회 — 설정 창에서 확인
내 혼디 코드는 언제든지 설정 창에서 꺼내볼 수 있습니다.
① 상단 바퀴 아이콘(⚙️) 탭
② 내 아이디·핸들 오른쪽에 혼디 코드 썸네일 표시
③ 썸네일 터치 → 크게 확대
④ "이미지로 저장" 버튼 → 스마트폰 갤러리에 저장

저장한 이미지를 인쇄소에 보내거나, 문서에 붙여넣어 바로 사용할 수 있습니다.
혼디 코드 부착 방법
저장한 이미지를 다양한 방식으로 활용하세요.
① 출입문 스티커 — 인쇄소에 이미지 파일을 주고 스티커로 제작합니다.

② 테이블 명함·메뉴판 — 명함 크기로 인쇄해 테이블마다 놓아두면, 손님이 혼디 앱으로 바로 스캔해 주문·결제할 수 있습니다.

③ 영수증·포장지 — 로고 옆에 작게 인쇄해두면 재방문 고객이 쉽게 등록할 수 있습니다.

④ 온라인 프로필·SNS — 이미지 파일을 인스타그램·카카오 프로필에 올리면 디지털 명함 역할을 합니다.

혼디 코드는 클수록 더 멀리서 인식됩니다. 부착 장소에 맞는 크기를 선택하세요.

대형 간판 (100cm) 약 10m 건물 외벽, 도로변 광고

혼디 코드 스캔 — 상대방의 코드 읽기
가게·명함의 혼디 코드를 스캔하면 상대방 프로필로 바로 이동합니다.
① 상단 카메라 아이콘(📷) 탭 — 전체화면 뷰파인더가 열립니다.

② "혼디" 글자를 화면 안에 맞추면 자동 인식 — 별도 버튼을 누를 필요가 없습니다. QR코드처럼 들이대기만 하면 됩니다.

③ 진동·소리 피드백 후 프로필 페이지로 자동 이동

조명이 어두울 때는 뷰파인더 상단 "📷 사진" 버튼으로 한 장 찍으면 분석해 드립니다.
혼디 코드 — 위변조 불가
혼디 코드는 GUID로부터 수학적으로 결정됩니다. 같은 번호를 가진 코드는 세상에 오직 하나뿐이며, 누군가 색상을 바꿔 위조하면 스캐너가 다른 사람의 프로필로 연결됩니다.
위조된 혼디 코드로 결제를 유도하는 사기를 원천 차단합니다. 스캔 후 연결된 프로필의 이름과 핸들이 실제 가게와 일치하는지 반드시 확인하세요.
무지개 막대 — "혼디"가 없어도 됩니다
혼디 코드는 사실 두 부분으로 나뉩니다. "혼ㄷ" 문자와 색상 막대(ㅣ)입니다. 지금까지는 둘이 항상 함께 붙어 있었지만, 색상 막대는 어떤 단어나 기호 옆에도 붙일 수 있습니다.
"중화요리" 간판 옆에 무지개 막대 → 중화요리집 프로필로 연결
"제주시청" 현판 옆에 무지개 막대 → 제주시청 공식 페이지로 연결
"제주지방법원" 표지판 옆에 무지개 막대 → 법원 안내 페이지로 연결

이 경우 스캐너는 색상 막대만 읽어서 프로필을 찾아줍니다. 단어 자체는 읽지 않습니다.
그러나 문제가 있습니다 — 스캐너는 어디를 봐야 하는가
"혼디" 코드에서 ㅎ(파랑)과 ㅗ(빨강)은 스캐너의 나침반 역할을 합니다. 이 두 색이 보이면 스캐너는 "아, 여기 색상 막대가 있구나"라고 인식하고, 색상 막대의 위치·크기·기울기를 정확히 계산합니다.
"중화요리" 간판에는 파랑·빨강 기준점이 없습니다.
스캐너 입장에서는 어디가 색상 막대인지 알 방법이 없습니다.

→ 새로운 "기준 부호(Anchor Mark)"가 필요합니다.
기준 부호 설계 — 3가지 후보 제안
색상 막대를 단독으로 쓸 때 스캐너가 인식할 수 있도록, 막대 양 끝(위·아래)에 작은 Anchor Mark를 붙입니다. 아래 세 가지 방식 중 하나를 선택하면 됩니다.
① 이중 사각형 (권장)
막대 위아래에 작은 정사각형 2개를 배치합니다. QR코드의 위치 탐지 패턴(□□)과 같은 원리입니다. 흑백 2중 테두리로 인쇄·조명 조건에 강합니다.
┌─┐
│■│ ← 위 앵커 (흑+백 이중 정사각형, 5×5px 기준)
└─┘
░░░
░░░ ← 색상 막대 (10~20칸)
░░░
┌─┐
│■│ ← 아래 앵커
└─┘ ② 컬러 도트 쌍 (간단)
막대 위 = 파랑 원형 도트, 아래 = 빨강 원형 도트. 현재 "혼디"의 ㅎ(파랑)/ㅗ(빨강)과 동일한 인식 방식을 재활용합니다. 기존 스캐너 코드 수정이 최소화됩니다.

③ L자형 코너 마크
막대 상단 좌우에 L자 모양의 검정 코너를 붙입니다. 기울어진 상태에서도 인식이 안정적입니다.
권장 방향 — ② 컬러 도트 쌍 채택
현재 혼디 스캐너는 이미 파랑 픽셀과 빨강 픽셀을 탐지하는 로직이 구현되어 있습니다. "혼디" 글자에서 ㅎ(파랑)과 ㅗ(빨강)을 찾는 코드입니다. 이 코드를 그대로 재활용할 수 있습니다.
범용 색상 코드(UCB) 구성:

🔵 ← 파랑 원형 도트 (지름 = 막대 폭의 1.5배)
████ ← 색상 막대 10칸 또는 20칸
🔴 ← 빨강 원형 도트
이 구성은 "혼디" 글자 없이도 스캐너가 동일하게 인식합니다.
파랑 도트가 위, 빨강 도트가 아래 — 이 방향이 v1/v2 판별과 색상 읽기 방향의 기준이 됩니다.
활용 시나리오 — 어디에든 붙일 수 있다
범용 색상 코드(UCB)를 사용하면 기존 간판·현판·명패를 그대로 유지하면서 색상 막대만 추가하면 됩니다.
가게 간판: "한림해녀국밥" 글자 오른쪽에 세로 방향 UCB 스티커 부착
공공기관: "제주시청" 현판 아래 또는 옆에 UCB 동판 매립
법원·병원: 기존 안내판 귀퉁이에 UCB 스티커 — 스마트폰으로 스캔하면 해당 기관의 혼디 프로필·서비스로 즉시 연결
명함·봉투: 이름 옆에 가로 방향 UCB 인쇄 (파랑 도트가 왼쪽, 빨강 도트가 오른쪽)
도로 표지판: 관광지·버스정류장 표지판에 UCB를 추가하면 스마트폰으로 해당 장소 정보 즉시 조회
기술 구현 — 스캐너 수정 범위
기존 스캐너(hondi-scanner.js)에서 수정해야 할 부분은 ROI 탐지 함수 하나입니다. "혼디" 글자 전체를 찾던 방식에서, 파랑 도트 + 빨강 도트 쌍을 찾는 방식으로 확장합니다.
현재: 파랑(ㅎ) + 빨강(ㅗ) + 종횡비 검증(혼디 글자 비율) → "혼디" ROI 확정

변경 후:
① 파랑 픽셀 클러스터 탐지 → 도트인지, ㅎ자모인지 판별
② 빨강 픽셀 클러스터 탐지 → 도트인지, ㅗ자모인지 판별
③-A 두 클러스터가 세로로 배열 + 사이 폭이 좁음 → UCB 모드 (범용 색상 코드)
③-B 두 클러스터가 "혼디" 글자 비율 → 혼디 코드 모드 (기존)

두 모드 모두 캘리브레이션과 색상 추출 로직은 동일하게 재사용합니다.
